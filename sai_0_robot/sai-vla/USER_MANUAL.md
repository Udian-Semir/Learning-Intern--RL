## 第一章 产品介绍

**SaiVLA-0** 是一款面向机器人操作任务设计的视觉—语言—动作（VLA）模型评测系统，主要用于在仿真环境中验证模型对自然语言指令的理解、对视觉场景的感知以及对机器人动作的规划与执行能力。系统以命令行客户端作为主要交互入口，能够将本地仿真环境（LIBERO）中采集的图像与机器人状态实时上传至云端推理服务，由模型完成任务理解与动作生成，再将预测出的动作序列回传至本地，驱动机器人完成抓取、放置、开合等原子操作，并以可视化视频和结构化指标的形式返回评测结果。

当前阶段，系统以 LIBERO 基准任务套件作为核心评测场景，用户通过一条命令即可完成从环境加载、指令下发、推理调用到结果回放的完整流程，从而直观体验机器人任务从「语言指令」到「分步执行」的全过程。

---

## 第二章 系统功能概述

SaiVLA-0 本质上是一个集**任务理解、视觉感知、动作规划与执行反馈**于一体的智能体系统。系统能够接收用户以自然语言描述的任务指令（例如「pick up the red mug」「put the bowl on the plate」），并结合机器人当前的摄像头视角与本体状态，对环境中的物体、空间关系及操作目标进行联合理解，进而生成一段可直接下发给机器人执行的连续动作序列。

在执行过程中，系统会按固定控制频率滚动更新观测信息，根据最新的视觉场景持续生成下一段动作，实现对任务进展的闭环跟踪。每一次评测以 **Trial（试次）** 作为最小运行单位，对输入图像、机器人状态、预测动作以及最终成败结果进行统一记录，并支持以视频和 JSON 摘要的形式集中输出，方便用户进行复盘、对比和归档。

围绕上述能力，系统对外提供以下核心功能：

- **多任务套件评测**：支持 `libero_spatial`、`libero_object`、`libero_goal`、`libero_long` 等多组基准任务。
- **云端推理对接**：本地仅负责仿真与可视化，推理计算由云端 GPU 服务器承担。
- **指标自动汇总**：自动统计任务成功率、步数、耗时等关键指标。
- **视频回放留档**：可选保存每个 Trial 的执行视频，便于直观查看效果。
- **Python API 调用**：除命令行之外，亦支持通过 Python 客户端在自定义流程中集成。

## 第三章 系统实现原理

SaiVLA-0 在整体设计上采用**「感知—推理—执行」分层协同**的智能体工作方式，将模型的高层语义理解与底层动作生成解耦，以获得更稳定的控制效果与更清晰的计算开销边界。

在运行过程中，系统首先由本地仿真环境采集当前场景的多路图像（如第三人称视角与腕部相机）以及机器人本体状态，并与用户输入的自然语言任务描述打包为一次标准请求；该请求经本地客户端发送至云端推理服务后，由模型完成**「视觉—语言联合编码 → 语义到控制映射 → 动作片段解码」**的完整流程，并以结构化的动作序列形式返回。随后，本地客户端按控制频率将动作逐步下发至仿真器，持续采集新一帧观测用于下一次推理，从而形成「观测—推理—动作—再观测」的闭环。

在技术实现上，系统支持文本与图像的多模态输入融合，以提升对真实场景任务的理解能力；同时通过固定格式的请求/响应协议对输入输出进行约束，确保动作序列在不同任务套件和不同机器人配置下保持一致；此外，系统在客户端侧内建了记录与回放机制，将每轮评测的输入、输出、视频与统计指标集中落盘，便于后续查阅与比较。从整体架构来看，SaiVLA-0 并非单一的脚本工具，而是由**本地仿真层、客户端交互层、云端推理层与结果归档层**协同组成的评测系统，其中客户端主要负责交互与展示，核心能力集中体现在模型侧的多模态理解与动作生成流程之中。

## 第四章 客户端功能说明

### 4.1 环境准备

使用本系统前，建议先准备一套干净的 Python 环境，以避免依赖冲突。推荐使用 Conda 创建独立环境：

```bash
conda create -n saivla python=3.10 -y
conda activate saivla
```

随后安装 LIBERO 仿真环境所需依赖：

```bash
pip install mujoco==3.3.7 robosuite==1.4.0 libero==0.1.1
```

环境准备完成后，即可进入客户端安装流程。

### 4.2 客户端安装

从开源仓库获取评测客户端源代码并以可编辑模式安装：

```bash
git clone https://github.com/saivla/saivla-0.git
cd saivla-0
pip install -e .
```

安装成功后，终端中将可使用 `sai0-eval` 命令。用户可通过 `sai0-eval --help` 查看全部可用参数。

### 4.3 服务连接检查

在正式发起评测之前，建议先确认客户端与云端推理服务之间的连通性。可通过以下命令执行一次连接检查：

```bash
sai0-eval --server https://api.synthoid.cloud --check
```

若返回连接成功提示，即表示当前网络环境可正常访问推理服务。若出现 SSL 或证书相关错误，可尝试将 `https://` 改为 `http://` 后重试。

### 4.4 API Key 配置与查看

公开预览阶段，系统对开发者开放免注册体验，**无需配置 API Key 即可直接发起评测**，便于用户快速上手体验模型效果。

未来在正式版本中，用户将通过注册账号的方式获取个人专属的 API Key，用于区分不同用户的请求与配额。届时，API Key 可通过命令行参数或环境变量传入客户端，建议用户获取后妥善保存，避免泄露或遗失。

### 4.5 任务套件选择

系统内置了四组标准 LIBERO 任务套件，覆盖空间推理、物体识别、目标理解与长时序规划等不同能力维度：

| 任务套件           | 能力侧重              |
| -------------- | ----------------- |
| libero_spatial | 空间关系理解（左/右/上/下等）  |
| libero_object  | 物体识别与属性区分（颜色、类别等） |
| libero_goal    | 任务目标与意图理解         |
| libero_long    | 长时序、多步骤复合任务       |

在执行评测命令时，通过 `--task-suite` 参数指定需要运行的套件名称即可。

### 4.6 发起评测任务

当环境与服务连接均就绪后，用户可通过一条命令发起完整的评测流程。例如，在 `libero_spatial` 套件上运行 10 个 Trial，并保存评测视频：

```bash
sai0-eval \
  --server https://api.synthoid.cloud \
  --task-suite libero_spatial \
  --trials 10 \
  --output-dir ./my_eval_results \
  --save-video
```

命令执行后，客户端将自动完成以下工作：

1. 加载对应的 LIBERO 任务套件；
2. 为每一个 Trial 初始化仿真环境与语言指令；
3. 持续向云端服务发送观测请求，并接收预测动作；
4. 在仿真器中逐步执行动作，直至任务完成或达到最大步数；
5. 将执行过程与结果写入指定的输出目录。

在运行过程中，终端会实时显示当前 Trial 的任务描述、进度条以及成功状态，便于用户直观跟踪评测进展。

> **提示：** 若运行过程中出现 LIBERO 相关的网络下载错误，可为当前会话配置本地代理后重试：
>
> ```bash
> unset ALL_PROXY all_proxy
> export HTTP_PROXY="http://127.0.0.1:10808"
> export HTTPS_PROXY="http://127.0.0.1:10808"
> export http_proxy="$HTTP_PROXY"
> export https_proxy="$HTTPS_PROXY"
> ```

### 4.7 查看评测结果

评测完成后，系统会在 `--output-dir` 所指定的目录下生成统一格式的结果文件，用于快速查看关键指标：

```bash
cat ./my_eval_results/summary.json
```

该摘要文件中包含本次评测的任务套件、Trial 数量、成功率、平均步数、总耗时等核心信息，便于进行不同配置、不同版本之间的对比分析。

### 4.8 视频回放与归档

若在发起评测时附加了 `--save-video` 参数，客户端会为每一个 Trial 同步录制执行视频，存放于输出目录下的 `videos/` 子目录中：

```bash
ls ./my_eval_results/videos/
```

用户可直接播放这些视频，直观查看机器人在每个 Trial 中的完整执行过程，用于：

- 验证模型在不同任务下的泛化能力；
- 发现失败 Trial 的具体失败原因；
- 作为演示素材或评测归档。

对于重要的评测结果，建议将整个 `output-dir` 目录整体打包并进行备份，以便后续复现与对比。

### 4.9 Python API 调用

除命令行之外，系统还提供了 Python 客户端库，便于用户在自定义流程中调用模型服务。典型使用方式如下：

```python
from sai0_vla_client import Sai0VLAClient

client = Sai0VLAClient("https://api.synthoid.cloud", api_key="sk-xxx")

actions = client.act(
    images=[agentview_img, wrist_img],   # np.ndarray, 形状为 (H, W, 3)
    state=[...],                         # 机器人本体状态向量
    instruction="pick up the red mug",   # 自然语言任务指令
)
```

调用方传入多路图像、机器人状态与任务指令，即可获得一段可直接下发执行的动作序列。该接口适用于将 SaiVLA-0 模型集成到自研仿真器、真实机器人平台或其他评测流水线中。

## 第五章 注意事项

为便于用户更好地使用本系统，在实际操作过程中应注意以下事项：

1. 运行评测前，务必先执行服务连接检查（`--check`），确认本地与云端推理服务之间的网络连通性。
2. 任务指令应尽量使用完整、准确、贴近原始 LIBERO 描述的语句，避免因描述过于简略或歧义而影响模型理解效果。
3. 若在自定义流程中传入图像，建议保持图像清晰、主体完整、分辨率与 LIBERO 采集视图一致，以获得更稳定的推理结果。
4. 建议为每一次独立的评测任务指定不同的 `--output-dir`，避免多次运行结果相互覆盖。
5. 对于重要的评测结果与视频数据，建议及时打包备份，以便后续对比与复盘。
6. 公开预览阶段无需 API Key 即可直接体验；后续正式版本启用鉴权后，请妥善保管个人 API Key，避免泄露。
7. 如遇到 LIBERO 相关资源下载失败或 SSL 证书错误，可参考第 4.3、4.6 节中的代理与协议切换建议重试。
