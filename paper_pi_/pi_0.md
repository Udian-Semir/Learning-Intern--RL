## 摘要

机器人学习有巨大的潜力来释放灵活、通用和灵巧机器人系统的全部能力，并解决人工智能中最深层次的问题。然而，将机器人学习提升到有效实际系统所需的通用性水平，面临着数据、泛化和鲁棒性方面的重大障碍。

在本文中，我们讨论通用机器人策略（即机器人基础模型）如何应对这些挑战，以及我们如何为复杂和高度灵巧的任务设计有效的通用机器人策略。我们提出了一种新颖的 **Flow Matching（流匹配）架构**，构建在预训练的视觉-语言模型（VLM）之上，以继承互联网规模的语义知识。

然后我们讨论了如何在来自多个灵巧机器人平台的大型多样化数据集上训练该模型，包括单臂机器人、双臂机器人和移动操作机器人。我们评估了模型通过直接提示执行任务、遵循来自人类和高级 VLM 策略的语言指令的能力，以及通过微调获取新技能的能力。我们的结果涵盖了广泛的任务，如叠衣服、清理桌子和组装盒子。

> **【小白解释】**
> **通用机器人策略 (Generalist Robot Policy)**：一个神经网络模型可以控制多种不同的机器人做多种不同的任务，而不是每种机器人每种任务分别训一个模型。"通用"的含义是跨机器人本体、跨任务类型。就像 GPT-4 可以写诗、写代码、翻译，π₀ 可以叠衣服、擦桌子、装盒子。
>
> **VLM (视觉-语言模型)**：能同时理解图像和文字的 AI 模型。给它一张图 + 一句话，它能理解两者的关联。例子：PaliGemma, GPT-4V, Qwen-VL。

## 一、引言

> *"人类应该能够换尿布、策划入侵、屠宰生猪、驾驶船只、设计建筑、写十四行诗、记账、砌墙、接骨、安慰临终者、接受命令、下达命令、合作、单独行动、解方程、分析新问题、铲粪、编程、做美味佳肴、高效战斗、英勇牺牲。专业化是昆虫的事。"* — Robert A. Heinlein

人工智能系统有各种形态和规模，从解决人类心智无法企及的复杂问题的高度专业化系统（如预测蛋白质构象），到能够根据文本提示生成逼真图像或视频的系统。然而，人类智能远超机器智能的轴心是**多功能性（versatility）**：在多样的物理环境中解决各种任务的能力，同时对环境约束、语言命令和意外干扰做出智能响应。

也许在 AI 中，朝着这种多功能性取得的最切实进展可以在大语言模型和视觉-语言模型中看到：这些系统在大规模、非常多样化的网络图文语料上预训练，然后用更精心策划的数据集微调以诱导期望的行为和响应模式。虽然这类模型已被证明展现出广泛的指令跟随和问题解决能力，但它们并未真正地**置身于（situated）** 物理世界中，它们对物理交互的理解完全基于抽象描述。

如果这类方法要为具备人类所拥有的物理多功能性的人工智能系统做出切实进展，我们需要在**物理数据**上训练它们——即来自具身机器人智能体的数据。

灵活且通用的模型具有巨大的实际影响，但也可能为当今机器人学习面临的一些最严峻挑战提供解决方案，如数据可用性、泛化和鲁棒性。在自然语言和计算机视觉领域，预训练在多样化多任务数据上的通用基础模型往往优于狭窄定制和专门化的解决方案。类似地，对于有效的专门化机器人系统，**先在高度多样化的机器人数据上预训练，再对目标任务微调或提示，可能更有效**。这可以解决数据稀缺的挑战，因为通用模型可以获得更多数据来源（包括来自其他任务、其他机器人甚至非机器人来源的数据），也可以解决鲁棒性和泛化挑战。

然而，开发这样的通用机器人策略面临若干重大挑战：
1. **规模**：大规模预训练的完整好处往往在小规模下不存在
2. **架构**：需要正确的模型架构，既能有效利用多样化数据源，又能表示与复杂物理场景交互所需的精细微妙行为
3. **训练配方（training recipe）**：正如 NLP 和 CV 中大规模模型依赖于精细的预训练和后训练数据策划策略

在本文中，我们提出了一个原型模型和学习框架，称为 **π₀**。

### π₀ 的三大设计

**第一，利用预训练 VLM 导入互联网规模的经验。** 通过将模型建立在 VLM 之上，我们继承了语言和视觉-语言模型的通用知识、语义推理和问题解决能力，然后进一步训练模型融入机器人动作，将其转化为 **VLA（Vision-Language-Action）模型**。

**第二，跨机器人本体训练（cross-embodiment training）。** 我们将来自多种机器人类型的数据合并到同一个模型中——单臂和双臂系统，以及移动操作机器人。不同机器人有不同的配置空间和动作表示，通过 zero-padding 和 image masking 统一处理。

**第三，Flow Matching 动作分块架构。** 为了执行高度灵巧和复杂的物理任务，我们使用动作分块架构配合 Flow Matching（一种扩散模型的变体）来表示复杂的连续动作分布。这使模型能够以高达 50Hz 的频率控制机器人完成灵巧任务。为了将 Flow Matching 与 VLM 结合，我们使用了新颖的 **action expert（动作专家）** 设计——在 VLM 的基础上增加一组独立权重专门处理动作 token。

> **【小白解释】**
> **Cross-embodiment training**：同时用 ALOHA 双臂机器人、Franka 单臂、UR5 单臂、移动操作机器人等的数据训练同一个模型。不同机器人的关节数不同（7维 vs 14维 vs 18维），但通过 zero-padding（补零）统一到最大维度，通过 mask 标记哪些维度是真实的。这是 π₀ 泛化能力的关键来源之一。

### 训练配方：预训练 + 后训练

与语言模型一样，我们的配方分为**预训练阶段**和**后训练阶段**：
- **预训练**：在非常大且多样化的语料上训练，使模型获得广泛的知识基础
- **后训练（微调/对齐）**：用更精心策划的高质量数据将模型专业化到下游应用

**为什么需要两个阶段？** 仅训练高质量数据，模型学不会从错误中恢复（因为高质量数据中很少出现错误）。仅训练低质量的预训练数据，模型学不会高效流畅地执行任务。两者结合提供期望的行为：模型尽量以高质量数据的方式行动，但仍保留了错误恢复和纠正的能力储备。

### 贡献

1. 基于 VLM 预训练和 Flow Matching 的新型通用机器人策略架构
2. 对这种机器人基础模型的预训练/后训练配方的实证研究
3. 在 10,000+ 小时机器人数据上的预训练，以及 20+ 个下游微调任务的评估
4. 展示了前所未有的灵巧性和通用性水平：叠衣服、清理桌子、把碗放进微波炉、叠鸡蛋、组装盒子、打包食品杂货等

## 二、相关工作

我们的工作建立在最近的大规模机器人学习方法以及多模态语言模型之上。

**与 VLA 模型的关系**：我们的工作与最近提出的 VLA 模型最为密切相关，这些模型使用预训练的 VLM 微调用于机器人控制（如 RT-2、OpenVLA）。这类模型使用**自回归离散化**来表示动作（将连续的动作值分成离散的 bin，像预测文本 token 一样预测动作 token）。相比之下，我们的模型采用新颖的设计，通过 **Flow Matching** 微调 VLM 来产生动作。这使我们能够处理高频动作分块（最高 50Hz）和高度灵巧的任务，而这也是先前自回归 VLA 的主要挑战。

> **【小白解释】**
> **自回归离散化 vs Flow Matching**：
> - **自回归离散化**（RT-2的方式）：把连续的关节角度分成 256 个"桶"（比如 -3.14 到 3.14 分成 256 个等间距区间），然后让模型一步步预测动作属于哪个桶，像预测下一个词一样。问题：毫米级的精度需要非常多的桶，而且预测 50 步的动作要跑 50 次模型，很慢。
> - **Flow Matching**（π₀ 的方式）：直接从连续空间采样，通过学习一个"速度场"把随机噪声逐步转换成精确动作。一次推理可以同时生成 50 步的动作，快且精度高。

**与扩散/Flow Matching 图像生成的关系**：像 Stable Diffusion 3 这样的图像生成模型已经证明了 Flow Matching 在高分辨率合成中的强大能力。我们的架构受到 **Transfusion** 的启发——用同一个 Transformer 处理多种目标，离散输出用交叉熵损失监督，连续输出用 Flow Matching 损失监督。在 Transfusion 的基础上，**我们发现为机器人特定的（动作和状态）token 使用一组单独的权重（action expert）能提升性能**——这类似于 Mixture of Experts 的设计。

> **【小白解释】**
> **Mixture of Experts (MoE)**：不是所有参数都处理所有 token。一部分参数（VLM backbone，2B）专门处理图像+文本 token，另一部分参数（Action Expert，300M）专门处理状态+动作 token。两组参数只在 self-attention 层交互。好处：机器人数据不会干扰 VLM 已经学好的视觉语言知识。

## 三、概述

我们的训练框架分三步：

**第一步：组装预训练混合数据集。** 包含我们自己的灵巧操作数据集（7 种机器人配置、68 种任务）和整个 OXE 数据集（22 种机器人的数据）。预训练阶段还使用多样化的语言标签，结合**任务名称**和**分段注释**（约 2 秒长的子轨迹细粒度标签）。

**第二步：预训练基础模型。** 使用 PaliGemma VLM 作为起点，用 Flow Matching 增加动作输出能力，在数据混合集上训练。基础模型展现出广泛的能力和泛化，但不一定在任何单一任务上达到最优性能。它可以响应语言命令执行各种任务，但熟练程度有限。

**第三步：后训练（微调）到下游任务。** 对于复杂和灵巧的任务，使用高质量策划数据适应模型。我们研究了小型数据集的高效后训练，以及大型数据集的复杂任务后训练（如叠衣服和移动操作）。

## 四、π₀ 模型（核心数学推导章节）

π₀ 模型主要由一个**语言模型 Transformer 骨干网络**组成。按照标准的 late fusion VLM 配方，图像编码器将机器人的图像观测嵌入到与语言 token 相同的嵌入空间中。我们进一步用机器人特定的输入和输出增强该骨干——即**本体感受状态**和**机器人动作**。

### 模型架构概览

π₀ 使用**条件 Flow Matching** 来建模动作的连续分布。Flow Matching 为我们的模型提供了高精度和多模态建模能力，使其特别适合高频灵巧任务。

受 **Transfusion** 的启发，我们使用一个 Transformer 但分配两组独立权重（或称**专家，expert**）：
- **VLM Backbone**（2B 参数）：处理图像 token + 文本 token，从 PaliGemma 初始化
- **Action Expert**（300M 参数）：处理状态 token + 动作 token，随机初始化

两组权重**只在 self-attention 层交互**——其他层（MLP、LayerNorm）各用各的。总参数约 3.3B。

> **【小白解释】**
> **两组权重如何在 self-attention 层交互？**
> 在每一层 Transformer 中：
> - **Self-Attention 层是共享的**：所有 token（不管来自 VLM 还是 Action Expert）的 Q、K、V 投影矩阵使用各自的权重，但 Attention 计算（Q×K^T → softmax → ×V）是在所有 token 间统一计算的
> - **MLP 层是分开的**：VLM backbone 的 token 走 VLM 的 MLP（宽度 2048），Action Expert 的 token 走 Action Expert 的 MLP（宽度 1024，更小更快）
>
> 这就像两个人（VLM 专家和动作专家）在开会讨论（self-attention），各自发表意见，但散会后各回各的办公室（MLP）干活。

### 问题形式化

我们想要建模数据分布 `p(A_t | o_t)`，其中：

- **动作块 (Action Chunk)**: `A_t = [a_t, a_{t+1}, ..., a_{t+H-1}]` — 未来 H 步动作（本论文 H=50）
- **观测 (Observation)**: `o_t = [I¹_t, ..., Iⁿ_t, ℓ_t, q_t]`
  - `Iⁱ_t`：第 i 个图像（2-3 个相机视角）
  - `ℓ_t`：语言指令 token 序列
  - `q_t`：关节角度向量（本体感受状态）

> **【小白解释】**
> **H=50 意味着什么？** 在 50Hz 控制频率下，H=50 对应 1 秒的动作序列。模型一次推理生成未来 1 秒的所有动作，然后在 0.5 秒后（执行 25 步）重新推理下一个动作块。这和 ACT 论文中 H=100 的思想完全一致，只是步数不同。

图像 `Iⁱ_t` 和状态 `q_t` 通过各自的编码器编码，然后通过线性投影层投影到与语言 token 相同的嵌入空间。

### Flow Matching 训练（核心公式推导）

对于动作块 `A_t` 中的每个动作，我们有一个对应的**动作 token**，通过 Action Expert 处理。训练时，我们使用条件 Flow Matching 损失监督这些动作 token。

> **【数学推导】Flow Matching Loss 的完整推导**

Flow Matching 的目标是学习一个向量场 `v_θ`，它定义了从噪声分布到数据分布的**概率路径**。

#### 第一步：定义概率路径

选择一个条件概率路径 `q(A^τ_t | A_t)`，即给定真实动作 `A_t`，在 Flow Matching 时间 τ 时的噪声动作分布：

```
q(A^τ_t | A_t) = N(τ·A_t, (1-τ)·I)
```

> **【小白解释】**
> - **τ (tau)**：Flow Matching 的时间参数，范围 [0, 1]
>   - τ = 0：纯噪声（均值 0，方差 1 的标准正态分布）
>   - τ = 1：真实动作（均值 = A_t，方差 = 0）
>   - 中间值：噪声和真实动作的插值
> - **为什么均值为 τ·A_t？** 当 τ=0 时均值为 0（纯噪声），τ=1 时均值为 A_t（真实动作），中间线性变化
> - **为什么方差为 (1-τ)·I？** 当 τ=0 时方差为 I（标准噪声），τ=1 时方差为 0（确定性），噪声逐渐消失

**与 sai_0/ACT 中公式的对比**：
- ACT：`x_t = (1-t)·ε + t·a`，此时 mean = t·a，var = (1-t)²
- π₀：`A^τ = τ·A + (1-τ)·ε`，此时 mean = τ·A，var = (1-τ)²

两者只是 τ 的定义方向不同（一个是从 0→1，另一个是从 1→0），数学上完全等价。

#### 第二步：定义目标向量场

沿着这条概率路径，存在一个最优的向量场 `u(A^τ_t | A_t)`，使得沿其积分能将噪声分布转换为数据分布。对于上述线性-高斯路径，最优向量场为：

```
u(A^τ_t | A_t) = A_t - ε
```

**推导**：从 `A^τ_t = τ·A_t + (1-τ)·ε` 对 τ 求导：

```
d(A^τ_t)/dτ = A_t + (d(1-τ)/dτ)·ε = A_t - ε
```

由于 (1-τ) 的导数为 -1，所以 `A_t - ε` 正是沿该路径的"速度"。

> **【数学推导】为什么是 "A_t - ε" 而不是 "ε - A_t"？**
>
> 从 `A^τ_t = τ·A_t + (1-τ)·ε` 出发：
> - 对 τ 求导：`d/dτ [τ·A_t + (1-τ)·ε] = A_t - ε`
>
> sai_0 中用的是 `x_t = t·a + (1-t)·ε` → 速度 = `a - ε`（与 π₀ 相同符号）
>
> 这和 π₀ 论文中表述的 `u = A_t - ε` 是一致的。**都是真实动作减去噪声**。

#### 第三步：训练目标

网络 `v_θ(A^τ_t, o_t)` 学习预测这个向量场：

```
L_τ(θ) = E_{p(A_t|o_t), q(A^τ_t|A_t)} [||v_θ(A^τ_t, o_t) - u(A^τ_t|A_t)||²]
```

其中 `||·||²` 是欧几里得距离的平方（即 MSE Loss）。

#### 第四步：实际训练过程

训练时，对每个样本：

1. 采样噪声：`ε ~ N(0, I)`
2. 采样时间：`τ ~ p(τ)`（从 Beta 分布的变体采样，强调低 τ）
3. 计算噪声动作：`A^τ_t = τ·A_t + (1-τ)·ε`
4. 计算目标：`u = A_t - ε`
5. 网络接收 `A^τ_t` 和 `o_t`，预测 `v_θ(A^τ_t, o_t)`
6. 计算 `MSE(v_θ, u)`

**Attention Mask**：Action Expert 使用**全双向注意力掩码**——所有动作 token 可以互相看到（不是因果/自回归的）。这很重要，因为 Flow Matching 去噪过程需要所有 token 之间的信息交换。

#### 第五步：时间步采样分布

不同于原始 Flow Matching 论文的均匀分布 `τ ~ U(0,1)`，π₀ 使用了一个特殊设计的时间步采样分布：

```
p(τ) = Beta((s-τ)/s; α=1.5, β=1.0)
```

其中 `s = 0.999` 是截止阈值。这个分布**强调低 τ（高噪声水平）**。

> **【小白解释】为什么强调低 τ？**
> 图像生成中，高 τ（低噪声）时任务很简单——网络只需学到恒等映射。但动作预测不同于此——**从观测预测平均动作本身就很难**（预测 E[A_t|o_t] 比预测"平均图像"难得多）。所以 π₀ 把训练资源集中在更难的"从高噪声中恢复动作"的任务上。
>
> sai_0 同样使用 `Beta(1.5, 1.0)` 分布（`t = (s-τ)/s` 的变换下等价）。

### 推理（Inference）

推理时，从 τ=0 的随机噪声开始，用学习到的向量场积分到 τ=1：

```
A⁰_t ~ N(0, I)                              # 初始化为纯噪声
A^{τ+δ}_t = A^τ_t + δ·v_θ(A^τ_t, o_t)      # 前向 Euler 积分
```

其中 δ 是积分步长。本文使用 **10 步积分**（δ = 0.1）。

> **【数学推导】Euler 积分的含义**
>
> 前向 Euler 法是解常微分方程 `dA/dτ = v_θ(A, o)` 的最简单数值方法：
> - 把 [0,1] 分成 10 份：τ₀=0, τ₁=0.1, ..., τ₁₀=1.0
> - 每步：`A_{k+1} = A_k + 0.1 × v_θ(A_k, τ_k, o)`
>
> **为什么只需要 10 步？** Flow Matching 学的是直线速度场，物理上只需 1 步（如果模型完全准确）。10 步是准确性和速度之间的平衡。sai_0 只用 4 步（因为 Action Head 更轻量，速度场更简单）。

**高效的推理实现**：推理时，我们缓存观测 `o_t` 的 attention keys/values（它们不随 τ 变化），只对动作 token 重复计算 10 次。图像编码器（SigLIP）每帧只需跑一次。

### Action Expert 架构细节

- **状态编码**：`q_t`（关节角度）→ 线性投影 → Transformer 嵌入维度
- **动作 + 时间步编码**：对于每个噪声动作 `a^τ_{t'}`，计算嵌入：
  ```
  embedding = W₃·swish(W₂·concat(W₁·a^τ_{t'}, φ(τ)))
  ```
  其中 φ(τ) 是正弦位置编码，将标量 τ 映射到高维向量。

> **【小白解释】这个公式在做什么？**
> - `W₁·a^τ`：把 32 维动作向量映射到 1024 维（Action Expert 的宽度）
> - `φ(τ)`：把时间步 τ（标量）用 sin/cos 函数扩展到 1024 维——让网络更好地感知当前处于去噪的哪个阶段
> - `concat(..., ...)`：把动作嵌入和时间嵌入拼接成 2048 维向量
> - `swish(W₂·...)`：非线性变换，合并动作信息和时间信息
> - `W₃·...`：再投影回 1024 维，得到最终的动作 token 嵌入
>
> 这和 sai_0 的 `MultiEmbodimentActionEncoder` 做的事情几乎一样，只是 π₀ 用 swish 激活而不是 nnx.swish，本质相同。

### Attention Mask 设计（分块因果注意力）

π₀ 使用**3 块分块因果注意力掩码**：

```
块 1: [I¹, ..., Iⁿ, ℓ]  ← 图像+语言 token（VLM backbone）
块 2: [q]                ← 状态 token（Action Expert）
块 3: [a^τ₀, ..., a^τ_{H-1}] ← 动作 token（Action Expert）
```

**规则**：
- 块内：全双向注意力（所有 token 可以互相看到）
- 块间：前面的块不能看到后面的块（块 1 看不到块 2、3；块 2 看不到块 3）
- 最后一块（动作）可以看到所有前面的块

**设计动机**：
1. 块 1（VLM token）不能看到块 2、3——这些是机器人特定的新输入，不让它们影响 VLM 的预训练表示（最小化分布偏移）
2. 块 2（状态）不和块 3（动作）交互——状态在去噪的 10 步中不变，可以缓存其 K/V 加速推理
3. 块 3（动作）可以看到所有前面的 token——需要利用视觉语言特征和状态来做去噪

### 非 VLM 基线模型：π₀-small

为了消融实验，我们还训练了一个不使用 VLM 初始化的较小模型（470M 参数）：
- 用 DistilBERT 代替 Gemma 做语言编码
- 用更小的预训练 ViT 编码图像
- Action Expert 使用 DiT 架构（AdaLN-Zero 层融入时间步），而非 Gemma 架构
- 使用编码器-解码器结构而非混合专家（MoE）结构
- 同样使用 Flow Matching（10 步推理）

## 五、数据收集与训练配方

### 预训练混合数据集

预训练混合数据集包含：
- **9.1%** 开源数据：OXE（22 种机器人）、Bridge v2、DROID（1-2 个相机，2-10Hz 控制频率，覆盖广泛的对象和环境）
- **90.9%** 自有数据：903M 个时间步，7 种机器人配置、68 种任务

**自有机器人平台（7 种）**：
| 平台 | 相机数 | 配置/动作维度 | 特点 |
|------|--------|-------------|------|
| UR5e 单臂 | 2 | 7 维 | 手腕 + 过肩相机 |
| 双臂 UR5e | 3 | 14 维 | 两个 UR5e |
| Franka | 2 | 8 维 | 7-DoF 臂 |
| 双臂 Trossen (ALOHA) | 3 | 14 维 | 基于 ALOHA 架构 |
| 双臂 ARX/AgileX | 3 | 14 维 | |
| 移动 Trossen/ARX | 3 | 16 维（+2 移动底座） | 基于 Mobile ALOHA |
| 移动 Fibocom | 3 | 17 维（+3 全向底座） | 全向移动底座 |

> **注意**：这里的"任务"定义和学术界的惯例不同。学术论文通常把"拿起杯子"和"拿起盘子"算作不同的任务。π₀ 的 68 个任务每个都是一个复合行为（如"清理桌子"包含识别+抓取+分类放置数十种不同物体），实际涵盖的行为范围远超"68 个任务"这个数字，比此前任何一个学术数据集的行为范围都要大几个数量级。

**数据处理**：
- 统一维度：配置向量和动作向量统一到整个数据集中最大机器人的维度（18 维 = 两个 6-DoF 臂 × 2 + 2 夹爪 + 2 移动底座 + 1 垂直躯干）
- Zero-padding：维度较小的机器人用 0 填充到 18 维
- Image masking：不足 3 个相机视角的机器人，缺失的相机位置用全零填充 + mask=False

**数据加权**：由于不同任务-机器人组合的数据量极不平衡（比如叠衣服任务的数据量远超其他任务），使用加权采样：`weight = n^{0.43}`，其中 n 是该组合的样本数——数据量越大的组合权重被压低。

### 语言和高级策略

对于需要语义推理和高级策略的复杂任务（如清理桌子），我们使用了一个**高级 VLM 策略**来将高层任务（如"清理桌子"）分解为中间子任务（如"拿起餐巾纸"、"把餐巾纸扔进垃圾桶"）。这类似于 SayCan 等 LLM/VLM 规划方法。

### 后训练（微调）

在后训练阶段，我们用较小但更高质量的任务特定数据集微调基础模型。不同任务需要的数据量差异很大：最简单的任务只需 5 小时的数据，最复杂的需要 100+ 小时。

## 六、实验评估

### 6.1 基础模型开箱评估

在 5 个任务上评估预训练后（不微调）的开箱表现：

| 任务 | π₀ (full) | π₀ (parity) | π₀-small | OpenVLA | OpenVLA (UR5e) | Octo |
|------|-----------|-------------|----------|---------|---------------|------|
| 叠 T 恤 | **~1.0** | ~0.9 | ~0.7 | ~0.0 | ~0.2 | ~0.0 |
| 清理桌子（简单） | **~1.0** | ~0.8 | ~0.6 | ~0.0 | ~0.2 | ~0.0 |
| 清理桌子（困难） | **~0.8** | ~0.6 | ~0.4 | ~0.0 | ~0.1 | ~0.0 |
| 打包杂货 | **~0.9** | ~0.7 | ~0.5 | ~0.0 | ~0.1 | ~0.0 |
| 从吐司炉取吐司 | **~0.9** | ~0.7 | ~0.5 | ~0.0 | ~0.0 | ~0.0 |

**关键发现**：
1. 即使训练步数相等（parity 版本），π₀ 也显著优于所有基线
2. OpenVLA 在这类任务上几乎完全失败——因为它的自回归离散化架构不支持 action chunking 和高频控制
3. π₀-small 虽然更小且没有 VLM 预训练，但仍然优于 OpenVLA 和 Octo——体现了 Flow Matching + action chunking 架构的优势

### 6.2 语言指令跟随

在 3 个语言条件任务上比较 π₀ 和 π₀-small：
- **清理桌子**：识别 12 种不同物体，分类放入碗篮或垃圾桶
- **摆桌子**：从篮子中取出物品按照语言指令摆好
- **打包杂货**：按要求将特定物品放入纸袋

π₀ 的语言跟随准确率显著优于 π₀-small，表明 VLM 预训练对语言理解能力有重大贡献。这种能力直接转化为更好的任务表现——无论是人类专家提供中间指令（π₀-human），还是高级 VLM 策略自主提供指令（π₀-HL）。

### 6.3 学习新的灵巧任务（微调）

在 5 个与预训练数据不同程度上不同的新任务上微调：

| 任务 | 难度 | π₀ (pretrained) | π₀ (scratch) | ACT | Diffusion Policy |
|------|------|----------------|-------------|-----|-----------------|
| 叠碗 | 容易 | **~0.9** | ~0.7 | ~0.6 | ~0.7 |
| 叠毛巾 | 容易 | **~0.9** | ~0.7 | ~0.5 | ~0.7 |
| 微波炉放保鲜盒 | 中等 | **~0.9** | ~0.8 | ~0.6 | ~0.5 |
| 换纸巾卷 | 困难 | **~0.7** | ~0.4 | ~0.3 | ~0.2 |
| Franka 物品放抽屉 | 困难 | **~0.7** | ~0.3 | ~0.1 | ~0.1 |

**关键发现**：
- 从预训练模型微调几乎总是优于从头训练（有时高达 2 倍提升）
- 预训练的好处随着任务难度增加而增大
- 有趣的是，最强的先前模型（ACT、Diffusion Policy）是在目标任务上**完全从头训练**的——这凸显了之前的方法在利用预训练方面的不足
- 随着微调数据量增加（1 小时 → 5 小时 → 15 小时），π₀ 的表现继续提升，而从头训练的 π₀ 提升较慢

### 6.4 掌握复杂的多阶段任务

最终实验在 8 个极其复杂的真实世界任务上评估，有些任务耗时 5-20 分钟完成：

| 任务 | π₀ (pretrain+finetune) | π₀ (out-of-box) | π₀ (scratch) |
|------|------------------------|-----------------|-------------|
| 叠衣服（固定底座） | **~0.85** | ~0.0 | ~0.2 |
| 叠衣服（移动底座） | **~0.75** | ~0.0 | ~0.1 |
| 烘干机卸衣 | **~0.80** | ~0.0 | ~0.1 |
| 清理桌子（真实午餐） | **~0.75** | ~0.1 | ~0.2 |
| 组装纸箱 | **~0.65** | ~0.0 | ~0.1 |
| 打包食物到外卖盒 | **~0.75** | ~0.0 | ~0.1 |
| 叠鸡蛋到蛋托 | **~0.70** | ~0.0 | ~0.1 |
| 卸干衣机 | **~0.80** | ~0.0 | ~0.1 |

**关键发现**：
1. 预训练对复杂任务至关重要——仅用开箱模型（out-of-box）几乎完全失败
2. 仅用高质量微调数据从头训练（scratch）表现也很差——模型会学会"看起来正确"的动作但无法从错误中恢复
3. 预训练+微调的组合实现了最佳表现：预训练提供了错误恢复能力，微调提供了任务熟练度

## 七、讨论、局限与未来工作

π₀ 是一个机器人基础模型的框架：在大规模多样化数据上预训练，然后开箱评估或微调到下游任务。我们的实证评估跨越了结合灵巧性、泛化性和长时间多阶段行为的任务。

### 与 LLM 训练范式的类比

我们的框架与 LLM 的训练程序类似：在网络上抓取的大规模数据集上预训练基础模型，然后通过后训练"对齐"模型使其能遵循指令和执行用户命令。普遍认为，LLM 的大部分"知识"是在预训练阶段获得的，而后训练阶段的作用是告诉模型**如何利用这些知识来满足用户命令**。我们的实验暗示，类似的现象可能也适用于机器人基础模型——预训练模型具有零样本能力，但复杂任务需要高质量数据微调。

### 局限

1. **数据组成**：我们还不能全面理解预训练数据集应该如何组成——哪些类型的数据更有帮助，如何加权，仍是开放问题
2. **可靠性**：并非所有任务都能可靠运行，如何预测需要多少数据才能达到接近完美的表现尚不清楚
3. **正迁移**：结合高度多样化数据（特别是不同任务和不同机器人）是否存在正迁移，仍有待观察
4. **通用性边界**：这种通用性能否扩展到更不同的领域（如自动驾驶、导航、四足运动）仍是开放问题

## 附录 B：模型架构详情

### 完整架构参数

**PaliGemma VLM Backbone (基于 Gemma 2B)**：
- `width=2048, depth=18, mlp_dim=16,384, num_heads=18, num_kv_heads=1, head_dim=256`
- 使用 Multi-Query Attention (MQA)
- ~2B 参数，从 PaliGemma 初始化

**Action Expert (基于 Gemma 300M)**：
- `width=1024, depth=18, mlp_dim=4,096`
- ~300M 参数，随机初始化

**模型总参数**: ~3.3B

### 额外输入输出

标准 PaliGemma 接收图像序列 + 语言提示。我们额外增加：
1. **状态输入 `q_t`**：通过线性投影映射到 Transformer 嵌入维度
2. **动作输入/输出**：50 个动作 token（H=50），每个动作 32 维（pad 到最大），通过 Flow Matching 以连续值生成

### Flow Matching 时间步的注入

对于每个噪声动作 `a^τ_{t'}`，嵌入公式为：

```
W₃ · swish(W₂ · concat(W₁ · a^τ_{t'}, φ(τ)))
```

其中：
- `W₁ ∈ R^{w×d}`：动作投影（d=32 动作维 → w=1024 嵌入维）
- `φ(τ) ∈ R^w`：正弦位置编码，将标量 τ 映射到 1024 维
- `concat` 产生 `2w = 2048` 维
- `W₂ ∈ R^{w×2w}`：2048 → 1024
- `W₃ ∈ R^{w×w}`：1024 → 1024
- `swish`：激活函数 `swish(x) = x·σ(x)`

### 推理延迟

在一张 NVIDIA RTX 4090 上：

| 操作 | 耗时 |
|------|------|
| 图像编码 (SigLIP) | 14 ms |
| 观测前向传播 | 32 ms |
| ×10 动作前向传播 (Flow Matching 去噪) | 27 ms |
| 网络延迟 (远端推理) | 13 ms |
| **总本地推理时间** | **73 ms** |
| **总远端推理时间** | **86 ms** |

### 动作执行策略

- UR5e 和 Franka (20Hz)：每 0.8 秒推理一次（执行 16 步后）
- 其他机器人 (50Hz)：每 0.5 秒推理一次（执行 25 步后）
- **不使用**时序集成 (temporal ensembling)——实验发现它反而降低了性能

## 附录 C：关键概念速查

| 概念 | 一句话解释 |
|------|-----------|
| **Flow Matching** | 学习一个速度场，通过积分将噪声逐步转化为精确动作 |
| **Action Chunking** | 一次预测 H=50 步动作，减少决策次数 |
| **Action Expert** | 独立的 300M 参数网络，专门处理状态和动作 token |
| **Cross-Embodiment** | 多种机器人数据统一训练，通过 zero-padding 和 mask 统一维度 |
| **预训练 vs 后训练** | 大规模多样化数据 → 基础能力；高质量策划数据 → 任务精通 |
| **Transfusion** | 同一 Transformer 同时处理离散（文本）和连续（动作）目标 |
| **PaliGemma** | Google 的 3B 参数开源 VLM，π₀ 的骨干网络 |
| **Mixture of Experts** | VLM backbone 和 Action Expert 两组权重分开，只在 self-attention 交互 |
| **前向 Euler 积分** | Flow Matching 推理时更新动作的数值方法：`A_new = A_old + δ × v_θ` |
