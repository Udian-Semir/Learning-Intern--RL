## 摘要

我们研究视觉-语言-动作 (VLA) 模型如何通过实际部署中的**强化学习 (RL)** 来改进。我们提出了 **RECAP**（RL with Experience and Corrections via Advantage-conditioned Policies，通过优势条件化策略的强化学习与经验和纠正），一个通过优势条件化对 VLA 进行 RL 训练的通用方法。

我们的方法将异构数据纳入自改进过程，包括演示、策略自主采集的数据，以及在自主执行过程中提供的专家遥操作干预。RECAP 首先用离线 RL 预训练一个通用 VLA，我们称之为 **π\*₀.₆**，然后通过真机数据采集将其专门化到下游任务，以达到高性能。

我们展示了用完整的 RECAP 方法训练的 π\*₀.₆ 模型可以在真实家庭中叠衣服、可靠地组装纸箱，以及使用专业意式咖啡机制作浓缩咖啡。在一些最困难的任务上，RECAP 将任务吞吐量提高了一倍以上，并将任务失败率降低约一半。

> **【小白解释】**
> **RECAP 的核心思想**：之前的版本（π₀、π₀.₅）用的是纯模仿学习（BC）——看人类演示数据，学着做。这种方法的问题是模型永远不会比人类演示做得更好，而且一旦犯了错，它不知道如何恢复（因为人类演示里几乎没有"犯错的场景"）。
>
> RECAP 用强化学习解决了这个问题：让模型自己跑任务，跑完之后根据"任务成没成功"给一个奖励信号。然后训练一个"裁判"（值函数，Value Function）来判断哪些动作是好的、哪些是坏的。最后用这个裁判的评分来改进模型，让模型多做"好"动作，少做"坏"动作。这和 AlphaGo 的自我对弈是同一个道理——不再只是模仿人类，而是通过自己犯错来自我改进。
>
> **吞吐量 (Throughput)**：每小时成功完成任务的次数。这既衡量了成功率（能不能做成），也衡量了速度（做得多快）。之前的模仿学习模型虽然成功率还行，但动作偏慢。RECAP 通过奖励"更快完成"的行为，让模型既快又准。

## 一、引言

> *"如果你不怕尝试，你能学到的东西是惊人的。"* — Robert A. Heinlein, *Have Space Suit–Will Travel*

熟能生巧：虽然人类在获取新技能时表现出非凡的灵活性，但精通总是需要通过反复尝试来学习。对于通用机器人基础模型（如 VLA 模型），我们可以通过提示灵活地为通用机器人指定任务。但就像人类一样，这些模型需要练习一项技能才能达到精通。这意味着不仅要利用**演示数据**，还要利用**自主采集的经验数据**——这些数据允许策略纠正在部署中实际犯下的错误，将速度和鲁棒性提高到超越人类遥操作的水平，并适应新的部署条件。

通过自主练习来学习的基础，在**强化学习 (RL)** 中被形式化，已有数十年的历史。但是，在一个通用的和可扩展的机器人学习系统中实例化这些原理面临着重大挑战：为大型模型设计可扩展和稳定的 RL 方法、处理来自不同策略的异构数据、以及在真实世界中设置带奖励反馈的 RL 训练——在真实世界中，奖励信号可能是模糊的或随机的。

> **【小白解释】**
> **为什么要在真实世界做 RL（而不只是仿真）？** 仿真中的 RL 已经非常成熟（AlphaGo、Dota 2、四足机器人行走），但机器人操作任务的仿真到真机迁移非常困难——布料如何变形、液体如何流动、光线在不同厨房中如何反射，这些都很难在仿真中精准建模。π₀.₆ 是在**真实机器人**上的 RL：让真机一遍遍尝试叠衣服，掉地上了就扣分，叠好了就加分。这需要极大的耐心和硬件成本（论文里说用了 3-4 台机器人并行采集数据）。这也是为什么论文的贡献不仅仅是算法，还有"如何让真机 RL 稳定运行"的工程方案。

在本文中，我们提出了 **RECAP**，一个使 VLA 模型能够在训练管道的所有阶段——从预训练一直到在自主执行数据上的训练——纳入奖励反馈的方法。RECAP 旨在通过一个结合演示、自主经验和专家干预的通用配方来解决这个问题。

从训练一个通用 VLA 并在许多不同机器人平台的多样化数据上训练的配方出发，RECAP 首先用**离线 RL** 预训练 VLA，然后通过在部署中收集的额外数据进一步训练。在这些部署中，机器人根据每次试次的结果接收（稀疏的）奖励反馈，并可能接收额外的专家干预来纠正错误。

训练过程遵循离线 RL 配方：我们训练一个**值函数**来评估任务完成进度，然后使用该值函数来估计数据集中每个动作的**优势**。通过以基于该优势的改进指示器为条件来训练策略，我们可以获得改进的策略。

> **【小白解释】**
> **离线 RL (Offline RL)** vs **在线 RL (Online RL)**：
> - **在线 RL**：智能体一边和环境交互，一边实时更新策略。问题：需要大量交互，真机跑不起。
> - **离线 RL**：先用一批已有的数据（不管是人类演示还是旧策略采集的）训练一个"裁判"（值函数），然后在这个数据集上改进策略。新策略再采集一批数据，加入数据集，反复迭代。
> - RECAP 用的是离线 RL 的迭代版本——"批数据→更新值函数→更新策略→采集新数据→重复"。这避免了在线 RL 的数值不稳定性，同时仍然能从新数据中学习。

我们可以使用 RECAP 来训练复杂任务的策略，例如叠各种衣物、组装纸箱，或制作浓缩咖啡饮品。我们在图 2 中展示了其中一些任务。

**训练流程**：方法首先在多样化多任务和多机器人数据集上用离线 RL 预训练 π\*₀.₆ 模型。π\*₀.₆ 是将 π₀.₆ 模型适配到 RL 的版本，而 π₀.₆ 是 π₀.₅ 的改进版本，增加了更大的 backbone 和更多样化的条件。π\*₀.₆ 增加了基于**二值化优势值**的条件能力，这使得纳入值函数来改进策略成为可能。

预训练之后，π₀.₆ 先通过微调将 π\*₀.₆ 模型适配到下游任务（使用演示），然后执行一轮或多轮的**真机数据采集**，用 RL 改进模型。用 RECAP 在自主经验上训练 π\*₀.₆ 将一些最难任务的吞吐量提高了一倍以上，并将失败率降低 2 倍或更多。

这使得 π\*₀.₆ 达到了实际有用的鲁棒性水平：我们能够让它在办公室连续 13 小时制作浓缩咖啡饮品，在新家庭中连续两个多小时叠新衣物且不间断，以及在工厂中组装用于真实包装的纸箱。

虽然 RECAP 基于在先前工作中已经探索过的独立算法组件，但这些组件的**特定组合**是新颖的，并且**结果首次**表明，一个带人类奖励反馈和干预的通用强化学习配方可以显著提高 VLA 模型在通过部署收集的经验下的鲁棒性和吞吐量。

## 二、相关工作

用模仿学习训练的策略已知会遭受**累积误差**问题，并且在最好的情况下也只能达到和演示数据一样好的表现。本工作的目标是超越离线演示的模仿学习，提高视觉-语言-动作策略的可靠性和速度。

先前的工作已经使用在线干预来改进机器人操作策略。我们采用了一种称为 **human-gated DAgger** 的干预形式。与这些工作相比，我们的方法使用**专家干预和完全自主的经验**，形成了一个整合多种数据源的基于 RL 的框架。

> **【小白解释】Human-gated DAgger**
> DAgger (Dataset Aggregation) 是模仿学习的经典算法：让策略跑一遍，人在关键帧纠正错误动作，把纠正后的数据加入训练集。Human-gated 的意思是人在"看策略跑"的过程中，只在必要时抢过控制权。π₀.₆ 的做法更进一步：不仅记录人的纠正，还记录机器人自己跑时的所有数据（即使没成功），然后用 RL 从这些"失败"的数据中学习。

有大量关于使用 RL 自主改进机器人操作策略的工作，包括使用基于 Diffusion 的策略的方法、在多任务设置中的方法，以及使用预训练多任务策略的方法。与这些工作不同，我们研究如何将真实世界 RL 扩展到大型 VLA 策略，用于长时间、精细操作任务。

许多最近的工作研究了如何通过 RL 改进基础 VLA 模型。一些工作直接应用 PPO 算法及其变体来微调 VLA，产生的方法难以以高效和可扩展的方式扩展到真实世界 RL。另一条研究路线探索了在预训练 VLA 模型之上的 RL 微调，其中 RL 要么训练残差策略，要么微调动作头网络，要么选择或精炼 VLA 提出的动作，要么优化作用于基于 Diffusion 的 VLA 的噪声空间的策略。其中一些工作也探索了将学习到的行为蒸馏回 VLA 以进行端到端迭代改进的方法。

这些先前的工作通常使用离散动作或简单的高斯连续动作分布。一个关键区别是，我们使用（迭代的）离线 RL 端到端地训练整个 VLA，并使用一个**表达性强的 Flow Matching VLA 模型**。这之所以可能，是因为一个简单且可扩展的**优势条件化策略提取方法**，该方法去除了在大型 VLA 模型上使用策略梯度风格目标的大部分复杂性。在我们的比较中，我们展示了这显著优于更传统的基于策略梯度的提取方案。

在方法论上与 RECAP 更密切相关的是，一些先前的工作已经整合了值函数和 VLA 在真实机器人上的端到端 RL 训练。例如，Huang et al. 将标定的 Q-learning 应用于离线演示数据集进行抓取任务，没有在线改进阶段。Zhang et al. 使用 DPO 从人类偏好优化拾取和放置技能，使用 VLA 的在线 rollout。Zhai et al. 和 Ghasemipour et al. 分别使用 PPO 和 REINFORCE 配合完成任务时间值函数来训练 VLA 执行移动碗、展开垫子和推动物体等任务。

与这些先前工作相比，我们描述了一个**用于 VLA 的迭代离线 RL 框架**，具有多重优势：
1. 我们的方法支持高容量 Diffusion 和 Flow-based VLA——不同于先前工作中研究的离散动作模型
2. 我们通过使用优势条件化策略提取方法来避免对在线 PPO 或 REINFORCE 的需求，该方法可以利用所有先前的（离策略或离线）数据
3. 我们的评估包括复杂、灵巧和长时间的任务，我们的方法在处理可变形物体、液体和多阶段任务时将吞吐量提高约 2×

先前的工作已经探索了基于奖励、价值和优势来条件化策略的想法，包括使用无分类器引导的方法。我们将这种方法扩展到预训练和微调大规模通用 VLA 策略，纳入各种数据源（包括演示、干预和自主策略 rollout）来学习真实机器人操作任务。

最近的研究还研究了如何有效训练多任务、语言条件的奖励函数和值函数。在这些工作的基础上，我们还训练了一个**语言条件的分布式值函数**，使我们能够为优势条件化 VLA 训练框架估计状态-动作优势。

## 三、预备知识

### 强化学习

我们考虑标准 RL 设定，其中一个智能体以策略 `π(a_t | o_t)` 的形式，根据观测 `o_t ∈ O` 选择动作 `a_t`。

我们将**轨迹**定义为 `τ = (o₀, a₀, ··· , o_T) ∈ O × A ··· O`。

**轨迹分布**由策略 `π(a_t | o_t)` 和随机动力学 `p(o_{t+1} | o_t, a_t)` 诱导：

```
ρ_π(τ) = p(o₀) ∏_{t=0}^{T-1} π(a_t | o_t) · p(o_{t+1} | o_t, a_t)
```

> **【数学推导】轨迹分布公式的含义**
>
> 从初始状态 `o₀` 开始的概率 `p(o₀)`，每一步：
> 1. 智能体根据当前观测 `o_t`，用策略 `π(a_t | o_t)` 选择动作
> 2. 环境根据动力学 `p(o_{t+1} | o_t, a_t)` 转移到下一个观测
> 3. 整个过程重复 T 次
>
> 整个轨迹的概率 = 初始概率 × 所有中间步骤的概率乘积。这就是"给定策略 π，产生轨迹 τ 的概率"。

**奖励函数**由 `r(o_t, a_t)` 给出（简记为 `r_t`），其中 `r_T` 是终止奖励。

定义**折现累积奖励**（即**回报**）为 `R(τ) = Σ_{t=0}^{T} r_t`（不使用折现因子，但可轻易添加，因为真实机器人任务不涉及无限时间 horizon）。

**RL 的目标**是最大化累积奖励（回报），学习一个策略最大化：

```
J(π) = E_{τ~ρ_π} [R(τ)] = E_{τ~ρ_π} [Σ_{t=0}^{T} r_t]
```

策略 π 的**值函数**定义为：

```
V^π(o_t) = E_{τ_{t+1:T}} [Σ_{t'=t}^{T} r_{t'}]
```

即：在状态 `o_t` 下，按照策略 π 继续行动，未来所有奖励的期望和。

> **【小白解释】V^π(o_t) 的直观含义**
> 值函数回答了"我现在在这个状态，如果继续按当前策略走，总共能拿多少分？"。V 值越高，当前位置越好。在 π₀.₆ 中，V 值被设计成"距离任务成功还有多少步"的负值——V 越接近 0，说明离成功越近。

动作 `a_t` 的**优势值**可以计算为：

```
A^π(o_t, a_t) = E_{ρ_π}[Σ_{t'=t}^{t+N-1} r_{t'} + V^π(o_{t+N})] - V^π(o_t)
```

这对应一个 N 步估计。

> **【小白解释】优势 (Advantage) 的含义**
> "这个动作 a_t 比我在状态 o_t 下做所有动作的平均效果好多少？"
> - 正优势 (+)：这个动作比预期好 → 值得增加概率
> - 零优势 (0)：正常动作，不偏不倚
> - 负优势 (-)：这个动作蠢，拖了后腿 → 应该避免
>
> N 步估计用接下来的 N 步真实奖励 + 第 N 步的值函数来估计当前动作的"真实贡献"，平衡了偏差（只看 1 步太短视）和方差（看整条轨迹噪声太大）。

### 正则化强化学习

与其最大化 `J(π)`，通常做法是在 RL 中使用正则化，优化一个在最大化奖励的同时**保持靠近某个参考策略 `π_ref`** 的策略。这在进行多步骤梯度更新而不采集新数据时尤其重要——在这种场景下，`π_ref` 通常对应收集训练数据的**行为策略**。

这可以通过目标来形式化：

```
J(π, π_ref) = E_{τ~ρ_π_θ} [Σ γ^t r_t] - β · E_{o~ρ_π_θ} [D(π(·|o) ‖ π_ref(·|o))]
```

其中 D 表示某种散度度量。

> **【数学推导】正则化 RL 目标的理解**
>
> 第一项：和标准 RL 一样，最大化累积奖励。
>
> 第二项：**惩罚偏离参考策略太远**——`D(π ‖ π_ref)` 衡量新策略和旧策略的距离。β 控制惩罚的强度。
>
> 为什么需要这个？想象你用同一批数据反复训练模型的多个 epoch。如果没有这个约束，模型会过度自信地只选"看起来好"的动作，而实际上那些动作可能只是运气好。正则化让模型保持谦逊——"即使看起来好，也不要离原始演示太远"。

对于 D 为 **KL 散度**的情况，有一个著名的结果——最优策略有封闭形式：

```
π̂(a|o) ∝ π_ref(a|o) · exp(A^{π_ref}(o,a) / β)
```

其中 β 为拉格朗日乘子。

**优势条件化策略改进定理**（RECAP 方法的核心基础）：

如果我们定义一个改进概率：

```
p(I | A^{π_ref}(o, a)) = g(A^{π_ref}(o, a)) / ∫ g(A^{π_ref}(o, a')) da'
```

其中 g 是单调增函数，且定义改进策略为：

```
π̂(a|o) ∝ π_ref(a|o) · p(I | A^{π_ref}(o, a))^β
```

则 **J(π̂) ≥ J(π_ref)** 保证成立。即：改进策略保证不比参考策略差。

> **【数学推导】这个定理为什么重要？**
>
> 它说：你不需要直接优化 RL 目标（那需要环境交互和梯度估计），只需要做两件事——
> 1. 训练一个值函数来估计 A^{π_ref}
> 2. 用优势值作为"过滤器"，高优势的数据多训练，低优势的数据少训练
> 3. 保证新策略比旧策略好（或至少不差）
>
> 这就是 **优势条件化 (Advantage Conditioning)** 的核心——把 RL 变成了一个带加权标签的监督学习问题。

## 四、RECAP：通过优势条件化策略的强化学习与经验和纠正

我们的方法包括以下步骤，可重复一次或多次来改进基础 VLA 模型：

**步骤 1：数据采集。** 在任务上运行 VLA，对每个 episode 用任务结果标签标注（决定奖励），并可选择性地提供**人类干预**来为早期迭代中的错误提供纠正示例。

**步骤 2：值函数训练。** 使用目前收集的所有数据训练一个大型多任务值函数（记为 V^{π_ref}），该函数能检测失败并判断任务完成所需的时间。

**步骤 3：优势条件化训练。** 为了用该值函数改进 VLA 策略，我们在 VLA 前缀中包含一个基于**优势值**的优化性指示器。这种"优势条件化"配方提供了一种简单有效的方法，用**次优数据**从值函数中提取更优策略。

**预训练阶段**：在整个预训练数据集上执行步骤 2 和 3——该数据集包含数万小时的演示，来自许多任务和多种不同的机器人。

**迭代改进阶段**：执行步骤 1、2、3 一次或多次，用自主采集的数据进一步改进 VLA。

### A. 分布式值函数训练

为了训练一个能在预训练或后训练阶段充当任何任务的可靠评判器的值函数，我们将 V^{π_ref} 表示为一个**多任务分布式值函数** `p_φ(V | o_t, ℓ)`，将观测 `o_t` 和语言命令 `ℓ` 映射到 B 个离散值桶上的分布。

> **【小白解释】分布式值函数 vs 普通值函数**
> 普通值函数输出一个标量："这个状态大概值 3.5 分"。分布式值函数输出一整个直方图："这个状态有 30% 概率值 2 分，45% 值 3 分，25% 值 4 分"。分布式方法更鲁棒，因为它捕获了值估计的**不确定性**——尤其对机器人在不同家庭中的差异化表现。

使用 `R_t(τ) = Σ_{t'=t}^{T} r_{t'}` 表示轨迹 τ 从时间步 t 到结束的**经验回报**，我们训练 `p_φ(V | o_t, ℓ)` 通过以下步骤：

首先将经验回报值 `R_t(τ)` 离散化到 B = 201 个桶中（用 `R^B_t` 表示离散化后的回报），然后最小化在当前数据集 D 中轨迹上的**交叉熵 H**：

```
min_φ E_{τ∈D} [ Σ_{o_t∈τ} H(R^B_t(τ), p_φ(V | o_t, ℓ)) ]   ... (1)
```

> **【数学推导】交叉熵训练值函数的含义**
>
> 这个公式和训练图像分类器（猫/狗）完全一样——只不过类别从 2 个变成了 201 个桶。模型输出一个 201 维的概率向量（每个桶代表一个值范围），交叉熵衡量"真实值落在桶 #57，模型认为它有多大可能在桶 #57"。最大化给定轨迹数据的似然 = 让模型在看到某个观测时，正确预测"这个状态的长期价值"。

这是对数据集 D 所代表的策略（即行为策略 `π_ref`）的值函数的**蒙特卡洛估计器**。预训练阶段 D = 人类演示，值函数捕获我们条件的任务和元数据的期望回报。后续迭代中，它偏向演示和学习策略回报的加权组合。

虽然这种"在线策略"估计器不如更经典的"离策略"Q 函数估计器最优，但我们发现它简单且高度可靠，同时仍然比模仿学习有实质改进。我们的方法可以在未来工作中扩展为容纳离策略估计器。

我们可以从学习到的值分布中提取连续值函数（以及优势）：

```
V^{π_ref}(o_t, ℓ) = Σ_{b∈[0,B]} p_φ(V=b | o_t) · v(b)
```

其中 `v(b)` 表示桶 b 对应的值。

### B. 通过优势条件化进行策略提取

一旦有了值函数 V^{π_ref}，我们需要一种使用该值函数来训练改进策略的方法——这称为**策略提取 (Policy Extraction)**。

在我们的设定中，有效的策略提取方法需要满足几个标准：

1. 有效利用**多样化离策略数据**——包括初始演示、专家干预，以及来自最新策略和旧策略的自主 episode。这与离线 RL 方法面临的挑战密切相关
2. **可扩展**，易于应用于大型 VLA 模型，包括使用 Flow Matching 或 Diffusion 生成动作的模型
3. 有效利用**好的（接近最优的）和坏的（次优的）数据**——如果我们想用自主经验改进策略，这很重要

> **【小白解释】为什么需要同时用好数据和坏数据？**
> 在纯模仿学习中，只有"好的演示数据"才有用，坏数据直接丢弃。但在 RL 中，即使是失败的 episode 也包含重要信息——"在那个状态下做了这个动作→导致失败→以后不要重复"。优势条件化通过给每个动作标上优势值（即使整条轨迹失败，某些中间步骤可能还是好的），让模型从所有数据中提取信号。

在现有的策略提取方法中，**策略梯度方法**（包括正则化策略梯度和重参数化梯度）可能是最广泛使用的。但这些方法难以应用于 Flow Matching 模型——Flow Matching 模型不容易提供可处理的**对数似然**，使得它们难以扩展到现代 VLA 架构。

另一种替代方案是使用**加权回归方法**，如 AWR（Advantage-Weighted Regression），它隐式提供对行为策略的正则化，并使用简单的（重要性加权的）监督学习目标。然而，这些方法丢弃或显著降低数据中很大一部分的权重——有效地实施一种**过滤模仿**技术。

我们选择的方案是使用**优势条件化**的变体——在此方案中，策略在所有数据上以监督学习方式训练，但增加一个额外的输入来指示基于优势的动作有多优化。这与文献中提出的多种方法密切相关——这些方法提出以轨迹的某个函数为条件来训练策略。

**核心推导（Bayes 规则应用）**：

我们方法中的特定公式与 CFGRL 最为接近。基于第三节中的公式，我们可以应用 **Bayes 规则**重写改进概率：

```
p(I | A^{π_ref}(o, a)) = π_ref(a | I, o) / π_ref(a | o)
```

将此应用到我们的设定，并加入语言条件，我们可以获得第三节中描述的正则化改进策略的另一种**封闭形式**：

```
π̂(a | o, ℓ) ∝ π_ref(a | o, ℓ) · [π_ref(a | I, o, ℓ) / π_ref(a | o, ℓ)]^β   ... (2)
```

> **【数学推导】这个封闭形式的直观理解**
>
> `π_ref(a | o, ℓ)` 是原始策略：不看优势值，在所有数据上训练。
>
> `π_ref(a | I, o, ℓ)` 是"知道好坏"的策略：只在好数据（I=True）上训练，在差数据（I=False）上训练时知道"这些是坏动作，不要学"。
>
> 两者的比值 `π_ref(a|I) / π_ref(a)` 衡量：看到优势信息后，对动作 a 的偏好多大程度超过不看优势信息时的偏好。这是"这个动作有多好"的信号。
>
> β 是放大系数：β=1 直接信任这个信号，β>1 放大信号（推理时更激进地选好动作），β<1 缩小信号（更保守）。

对于特例 β = 1：**π̂(a | o, ℓ) = π_ref(a | I, o, ℓ)**。

因此，我们可以在**不显式表示改进概率 p(I | A^{π_ref}(o, a))** 的情况下表示 π̂——如果我们训练策略使其能够同时表示 `π_ref(a | o, ℓ)` 和 `π_ref(a | I, o, ℓ)`。

这个原理类似于**无分类器引导 (Classifier-Free Guidance, CFG)** 的方法——其中一个 Diffusion 模型被训练来同时建模"有"和"没有"条件变量的数据。

我们假设改进指示器 I 遵循**delta 分布**：

```
p(I | A^{π_ref}(o, a, ℓ)) = δ(A^{π_ref}(o, a, ℓ) > ε_ℓ)
```

其中 `ε_ℓ` 是任务相关的改进阈值。这个阈值允许我们控制优化性指示器，并最小化在训练后寻找衰减因子 β 来锐化改进条件分布的需要。

> **【小白解释】ε_ℓ 如何确定？**
> 预训练时：ε_ℓ = 第 30% 百分位的值（即 30% 的数据有正优势）。这个比例是经验选择的——太高（如 50%）会让太多数据被标为"差"，策略无法学到正常行为；太低（如 5%）会让几乎全部数据标为"好"，和直接用 BC 没区别。
>
> 微调时：ε_ℓ = 第 40% 百分位。更严格的选择确保模型在微调阶段被推向"更好"的方向。

**最终训练目标**为最小化以下**负对数似然**：

```
min_θ E_{D_{π_ref}} [ -log π_θ(a_t | o_t, ℓ) - α · log π_θ(a_t | I_t, o_t, ℓ) ]
其中 I_t = 𝟙[A^{π_ref}(o_t, a_t, ℓ) > ε_ℓ]   ... (3)
```

> **【数学推导】两个对数似然项分别的作用**
>
> **第一项** `-log π_θ(a_t | o_t, ℓ)`：无条件的模仿学习——在所有数据上，不管好坏，都尽量匹配。这是"保持基本能力"的项。
>
> **第二项** `-α · log π_θ(a_t | I_t, o_t, ℓ)`：有条件的学习——只在 I_t=True 的数据上强匹配（好动作多学），在 I_t=False 的数据上弱匹配（坏动作忽略）。α 控制"追求优秀"相对于"保持稳定"的权重。
>
> 两项一起优化，模型既不会忘记基本能力（第一项），也不会盲目模仿所有数据（第二项过滤了差数据）。这就是优势条件化把 RL 转化为监督学习的数学本质。

优势值 `A^{π_ref}(o_t, a_t, ℓ)` 从上一节的值函数中获得，α 是权衡超参数。

实践中，数据集 D_{π_ref} 包括到目前为止收集的所有数据（所有演示和自主任务尝试），参考策略 π_ref 因此是人类行为和先前部署策略的**混合**。

**对人类纠正的特殊处理**：我们发现强制对在自主 rollout 期间由专家纠正产生的动作设置 `I_t = True`（即正面）是有用的。如果我们假设专家纠正者总是提供好的动作，这个选择是合理的。

在实践中，我们的 VLA 模型产生**离散和连续输出**——连续分布通过 Flow Matching 表示。因此，真实的训练目标结合了离散值的似然和连续值的 Flow Matching 目标。

在实践中，我们在整个预训练数据集上**预训练一个模型**来表示 `π_θ(a_t | I_t, o_t, ℓ)`，然后对每个任务**执行一轮或多轮**带策略自主 rollout（和可选的专家纠正干预）的迭代方法。

### C. 方法总结

我们在算法 1 中提供了完整方法的概览。正如本节开头所总结的，该方法可以通过三个子程序的调用来完全定义：

1. **通过自主 rollout 收集数据**（带可选的专家纠正干预）
2. **根据公式 (1) 训练值函数**
3. **根据公式 (3) 训练策略**

方法不同步骤之间唯一变化的是提供给每个子程序的数据：**预训练阶段**使用所有先前的演示数据，而每个技能 ℓ⁽ⁱ⁾ 的**专门化训练**使用额外的自主数据。

实践中，专门化模型从预训练模型微调，而最终通用模型从头训练。

## 五、实现、模型与系统细节

我们用称为 **π\*₀.₆** 的 VLA 来实例化 RECAP。

### A. π₀.₆ 模型

π₀.₆ 模型源自 π₀.₅ 模型，能够通过 Flow Matching 灵活表示块化动作分布，并为高层策略推理产生中间文本。

它使用 **Knowledge Insulation (KI)** 训练程序——端到端训练整个模型，同时处理连续动作和离散 token（包括通过 FAST 离散化的动作），并使用 **stop gradient（停止梯度）** 防止 Flow-Matching Action Expert 影响模型的其余部分。预训练同时使用机器人数据和来自网络的视觉-语言联合训练数据。

> **【小白解释】Knowledge Insulation 和 stop gradient**
> KI 是保护 VLM 通用知识不被机器人数据"污染"的训练技巧。stop gradient 意味着：当 VLM backbone 处理图像和文本时，Action Expert 的梯度**不会反向传播**到 VLM backbone。这样 VLM 始终保留着从互联网学到的通用视觉和语言能力，而不会被机械臂的特定动作分布"带偏"。π₀.₆ 通过独立训练值函数和策略来实现类似效果。

**π₀.₆ 相对于 π₀.₅ 的改进**：

1. **更多预训练数据**：从多个机器人平台添加了额外数据
2. **基础 VLM 升级**：从 PaliGemma 2B 升级到 **Gemma 3 4B**
3. **Action Expert 扩大**：参数从 300M 增加到 **860M**

> **【小白解释】为什么 Action Expert 从 300M 扩到 860M？**
> π₀.₆ 的任务（叠各种衣物、做咖啡）对动作精度和速度的要求远高于 π₀.₅（清理桌子）。更大的 Action Expert 能更精确地建模复杂的连续动作分布。这和更大 VLM 的关系不同——VLM 负责"理解场景"，Action Expert 负责"做动作"。如果动作本身需要更精细（倒咖啡 vs 移动杯子），就需要更大的动作专家网络。

**模型表示**：可以写为 `π_θ(a_{t:t+H}, ℓ̂ | o_t, ℓ)`，其中：
- `o_t = [X¹_t, ..., Xⁿ_t, q_t]`：相机图像和关节配置
- `ℓ`：整体任务提示 + 额外元数据
- `ℓ̂`：预测的下一子任务（如"拿起咖啡杯"）
- `a_{t:t+H}`：50 Hz 的关节角度和夹爪命令块

动作使用独立的 **Action Expert**（860M 参数，用 Flow Matching 专门训练），但可以注意到模型中其余部分的激活。模型也产生 token 化的离散输出 `ℓ̂`——包含预测的子任务的文本表示。

**训练似然分解**：

```
log π_θ(a_{t:t+H}, a^ℓ_{t:t+H}, ℓ̂ | o_t, ℓ) =
    log π_θ(ℓ̂ | o_t, ℓ)                          # 子任务预测
  + log π_θ(a^ℓ_{t:t+H} | o_t, ℓ, ℓ̂)            # 离散动作预测 (FAST)
  + log π_θ(a_{t:t+H} | o_t, ℓ, ℓ̂)              # 连续动作预测 (Flow Matching)
```

### B. 从 π₀.₆ 到 π\*₀.₆：优势条件化

为了将优势信息纳入策略，我们扩展模型输入以包含一个额外的**改进指示器**——以额外文本输入的形式：
- `I_t = True` → 输入 "Advantage: positive"
- `I_t = False` → 输入 "Advantage: negative"

VLA 模型在其它方面与 V-A 节中描述的相同。优势指示器在训练序列中出现在 `ℓ̂` 之后，但在（离散化和连续的）动作之前——因此只有动作的对数似然受到影响。

连续部分的对数似然无法精确评估，而是通过 **Flow Matching 损失**来训练。可以在 Flow Matching 和 Diffusion 之间建立紧密的平行关系（在某些假设下），而后者又可以解释为对数似然的下界——因此我们可以大致将离散动作的对数似然 + 连续动作的 Flow Matching 损失之和作为整体动作似然的下界：

```
log π_θ(a_{t:t+H}, a^ℓ_{t:t+H} | I_t, o_t, ℓ, ℓ̂) ≥
E_{η,ω}[ log p_θ(a^ℓ | I_t, o_t, ℓ, ℓ̂)
         - α_η · ||ω - a - f_θ(a^{η,ω}, I_t, o_t, ℓ, ℓ̂)||² ]   ... (4)
```

其中 `a^{η,ω}_{t:t+H} = η·a_{t:t+H} + (1-η)·ω`，ω ~ N(0,I)，η ∈ [0,1] 是 Flow Matching 时间索引，f_θ 是 Action Expert 的连续输出，α_η 是损失权重项（可选的噪声相关）。

**训练时随机丢弃 I_t**：我们随机丢弃指示器（30% 丢弃率），替代了调整损失乘数 α 的需求。这使我们能够要么直接从带 `I_t = True` 的策略采样（对应公式 (2) 中 β = 1），要么同时使用条件和非条件模型来实现**无分类器引导 (CFG)**——这使得能够以 β > 1 进行推理。

> **【小白解释】CFG (Classifier-Free Guidance) 在 VLA 中的应用**
>
> 在图像生成中，CFG 使得 Stable Diffusion 能生成"更像文本描述"的图像——通过放大"有文本条件"相对于"无文本条件"的梯度。
>
> 在 VLA 中，CFG 被用来放大"优势条件化"的效果。训练时的 30% 随机丢弃确保模型同时学会了 π_θ(a|I, o)（知道好坏）和 π_θ(a|o)（不知道好坏）两个版本。推理时，通过 β>1 的组合，模型可以被推向"更优化"的动作方向——即使这些动作在训练数据中不那么常见。这就是"让模型比演示数据更强"的最后一个环节。

### C. 奖励定义与值函数训练

因为我们的目标是开发一个通用且广泛适用的方法来从经验中训练 VLA，我们使用了一个可以应用于几乎任何任务的通用稀疏奖励定义。

**对于每个 episode，我们获得一个标签指示该 episode 是否成功**。我们从这个 episode 级别的成功标签中推导奖励，使得值函数对应（负的）直到 episode 成功完成的步数。

这等价于以下奖励函数（T 对应 episode 的最后一步，C_fail 是一个大常数，确保失败 episode 有低值）：

```
r_t = 0,          若 t = T 且成功
      -C_fail,     若 t = T 且失败
      -1,          否则
```

> **【数学推导】为什么用"负步数"设计？**
>
> 每一步"正常进行"时 r_t = -1 → V 值每步下降 1。所以成功到达终点 (r_T = 0) 的轨迹：V(o₀) = -(T-1) ≈ -成功步数。失败的轨迹：V(o₀) = -(失败时步数) - C_fail，值非常低。
>
> 两个好处：
> 1. **自然区分快慢**：同样成功，步数越少 V 值越高→策略被激励做"更快的成功"
> 2. **对失败敏感**：成功和失败的值之间有 C_fail 的巨大差距→模型明确知道什么动作导致失败
> 3. **不需要人类定义每步的稠密 reward**（如"距离物体近了 0.1 分"）→只需要最终成功/失败的标签

在实践中，我们将预测值标准化到 (-1, 0) 之间。由于我们在不同长度的多样化任务上训练（清洗厨房 10 分钟 vs 叠衣服 2 分钟），我们根据每个任务的最大 episode 长度对值进行归一化。

值函数接收与 π\*₀.₆ VLA 相同的语言输入，并使用相同的架构设计——但有一个**更小的 670M 参数 VLM backbone**，也从 Gemma 3 初始化。为防止过拟合，我们还用一小部分多模态网络数据联合训练值函数。

### D. 预训练、数据采集和从经验中学习

我们模型预训练阶段中使用的数据混合物基本遵循 π₀.₅ 使用的配方——包括来自网络的视觉-语言数据、子任务 `ℓ̂` 的预测，以及来自许多不同机器人的各种低层动作任务的预测。

需要注意的是，预训练后，π\*₀.₆ 可以执行比第六节评估中使用的多得多的任务。

在预训练期间，我们首先在相同数据集上训练值函数，预测（负的）每个任务成功完成的步数。然后我们估计**每个任务的改进阈值 ε_ℓ**——设定为值函数对任务 ℓ 预测的值的第 30% 百分位数。然后在 VLA 训练期间**动态运行值函数**来估计每个样本的 A^{π_ref}(o_t, a_t, ℓ)，并用它来计算基于 ε_ℓ 的 I_t。I_t 作为输入包含到 π\*₀.₆ 中。

由于值函数使用相对较小的 VLM backbone (670M)，在 VLA 训练期间的动态推理仅产生最小的额外成本。

**预训练后的策略改进循环**：

1. 首先用目标任务 ℓ 的演示数据 D_ℓ 微调 π\*₀.₆（此阶段固定 I_t = True，这对应**监督微调 SFT**）
2. 产生初始策略 π⁰_ℓ，然后用它收集额外数据加入 D_ℓ
3. 一些 episode 完全自主收集，另一些由专家遥操作员监控——操作员可以干预提供纠正
4. **纠正可以展示如何避免灾难性失败或如何从错误中恢复**
5. 注意：仅靠纠正不太可能解决所有问题——干预自主执行是破坏性事件，即使是专家人类操作员也不能保证干预的一致质量，也不能改进微妙的方面（如总体速度）
6. 纠正更多是为了修复大错误和克服探索挑战，本身不提供最优监督

**从人类纠正部分强制 I_t = True**——但对 episode 的其余部分（无论是否提供纠正），整个 episode 可选添加到 D_ℓ 中。

**数据收集后**：在所有收集的任务数据上微调值函数，然后用更新的指示器 I_t（使用相同程序）微调策略。**值函数和策略都从预训练的 checkpoint 微调，而不是从上次迭代的模型**——我们发现这有助于跨多轮迭代避免漂移。

这个过程可以根据需要重复多轮，但在实践中我们发现即使一轮也常常带来显著改进。

## 六、实验评估

在我们的实验评估中，我们使用 RECAP 在一组真实任务上训练 π₀.₆ 模型：制作浓缩咖啡饮品、叠各种衣物和组装纸箱。

**每个任务需要多个步骤（持续 5 到 15 分钟），复杂的操作行为（约束力操作、倒液体、操作布料和纸板等），以及快速执行以提供高吞吐量。**

### A. 评估任务

**① 叠衣物（T恤和短裤）**：π₀ 论文中的标准叠衣任务。从篮子中取出一件 T 恤或短裤（初始条件可变），摊平、折叠。成功标准 = 一件衣物折叠并堆放在桌子右上角，200 秒内。

**② 叠衣物（多样化物品）**：折叠更多样化的衣物——11 种类型（毛巾、纽扣衬衫、毛衣、牛仔裤、T 恤、短裤、Polo 衫、裙子、长袖衬衫、袜子、内衣）。成功 = 目标物品正确折叠并放在桌子上堆叠，500 秒内。

**③ 叠衣物（定向去故障）**：更具结构化的版本——只叠一件橙色 T 恤，初始条件固定为摊平。成功标准严格（领口始终朝上，200 秒内）。用于评估 RECAP 是否能通过 RL 消除特定不良行为。

**④ 咖啡厅（双份浓缩）**：用商业意式咖啡机制作双份浓缩。步骤：拿起手柄 → 放到研磨机上研磨咖啡豆 → 压实咖啡粉 → 手柄锁入咖啡机 → 拿杯 → 萃取完整双份 → 服务。成功 = 200 秒内完成所有步骤且无重大错误。

**⑤ 组装纸箱**：真实工厂部署场景——从摊平的纸板开始折叠纸箱，贴上标签，放进箱子中的适当位置。成功 = 600 秒内从摊平到组装好并堆叠。

### B. 对比和消融

比较以下模型：

- **预训练 π₀.₅**：无 RL，无 RECAP
- **预训练 π₀.₆**：不含优势指示器 I_t，用监督学习预训练
- **RL 预训练 π\*₀.₆**：带值函数一起预训练，包含 I_t
- **π\*₀.₆ offline RL + SFT**：从基础 checkpoint 微调（I_t 固定 True）
- **π\*₀.₆ (Ours)**：用 RECAP 训练的最终模型（含自主 rollout + 专家纠正）

还比较两种替代策略提取方法：**AWR** 和 **PPO**。

### C. 定量结果

**评估指标**：吞吐量（每小时成功次数）和成功率。

**全面性能提升**：π\*₀.₆ 在所有任务上显著超越 π₀.₆ 基础模型、RL 预训练模型和 offline RL + SFT 模型。在多样化叠衣和浓缩咖啡任务上吞吐量翻倍，失败率约减半。

**多轮迭代改进**（T恤+短裤任务、组装纸箱）：两轮迭代带来稳定改进。叠衣任务吞吐量总体提升 50%，组装纸箱第二迭代吞吐量翻 2×。叠衣任务第一轮就将成功率提到 90%+，第二轮主要改善速度。

**策略提取方法对比**（AWR vs PPO）：RECAP 显著优于 AWR 和 PPO。PPO 需要小的 trust-region 约束 (η=0.01) 来稳定训练，但即便稳定也不能达到好性能。AWR 能达到合理成功率但策略慢得多，吞吐量低。

**故障模式消除**（领口朝上叠衣，严格的成功标准）：应用 RECAP 两轮迭代（每轮 600 条轨迹）→ 策略以 97% 成功率高速运行。证明 RECAP 即使在完全无干预数据/额外演示的情况下也能消除特定失败模式。

## 七、讨论与未来工作

训练能够在真实世界任务上达到和人类一样的鲁棒性、速度和流畅性的策略，是机器人学习中的重大挑战。在本文中，我们讨论了**通过结合 DAgger 风格的教练指导和 RL 来从经验中学习**如何开始应对这一挑战。

我们描述了 **RECAP**，一个用自主试次、奖励反馈和人类干预训练 VLA 的方法，并展示了用 RECAP 训练的模型 π\*₀.₆ 在一组真实任务上的结果。RECAP 的核心是一个适合 VLA 策略可扩展训练的 RL 方法——使用**优势条件化**进行策略提取，配合值函数。该 RL 方法的数据通过自主 rollout 和人类干预的组合收集——用干预纠正错误，同时用自主数据微调行为细节。

我们的实验表明 RECAP 可以提高 VLA 的成功率和吞吐量，在一些较难任务上吞吐量提高了一倍以上，失败率降低约 2×。

### 未来工作方向

1. **系统不是完全自主的**：依赖人类标注、干预和 episode 重置。先前的工作已经探索了自动化这些组件的方法，VLA 提供了新的自动化数据收集方式（如使用高层策略推理场景重置）
2. **探索相对 naive**：主要依赖策略的随机性和人类干预来探索新解决方案。当初始模仿学习策略已经采取合理动作时这是合理的，但使用更复杂的探索方法有很大改进空间
3. **迭代离线更新而非完全在线 RL**：出于便利性选择离线批更新（收集一批数据，重新训练，重复），扩展到完全并发的在线 RL 框架是有前途的方向

**更广泛地**，用 RL 训练 VLA 可能是达到适合真实世界用例性能水平的最直接路径。VLA 的 RL 面临诸多挑战——从高容量模型的大规模 RL 训练的难度到样本复杂性、自主性和延迟反馈。虽然为较小规模系统或"虚拟"领域（如 LLM）设计的现有 RL 框架可以提供一个好的起点，但需要更多研究来使 RL 成为 VLA 训练的实用工具。我们希望我们的工作代表了这个方向上有意义的一步。

## 核心公式速查

| 公式 | 含义 |
|------|------|
| `V^π(o) = E[Σ r_t]` | 值函数：未来累积奖励的期望 |
| `A^π(o,a) = R - V(o)` | 优势：这个动作比平均好多少 |
| `π̂ ∝ π_ref · exp(A/β)` | 正则化 RL 的最优解 |
| `π̂ ∝ π_ref · [π(a|I)/π(a)]^β` | 优势条件化的封闭形式 |
| `I = 𝟙[A > ε]` | 二值化优势指示器 |
| `L = H + α·\|\|ω-a-f^a\|\|²` | Flow Matching 联合损失 |
| `r_t = 0/-C_fail/-1` | 通用稀疏奖励（成功/失败/进行中） |

## 关键概念速查

| 概念 | 解释 |
|------|------|
| **RECAP** | RL + 经验 + 纠错：通过优势条件化让 VLA 从部署中自我改进 |
| **值函数 (V)** | "裁判"：评估当前状态有多好 |
| **优势 (Advantage)** | "这个动作比平均水平好多少" |
| **离线 RL** | 在已有数据上迭代改进，无需实时环境交互 |
| **分布式值函数** | 输出值的概率分布（而非标量），更鲁棒 |
| **CFG** | 无分类器引导：调节 β 控制策略优化性 |
| **Human-gated DAgger** | 人在策略执行时只在必要时干预 |
