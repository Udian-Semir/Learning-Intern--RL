## 摘要

我们研究 VLA 模型如何通过实际部署中的**强化学习 (RL)** 改进。提出 **RECAP**（RL with Experience and Corrections via Advantage-conditioned Policies）——通过优势条件化对 VLA 进行 RL 训练的通用方法。

RECAP 将异构数据纳入自改进过程，包括演示、策略自主采集数据和专家遥操作干预（在自主执行过程中纠正错误）。首先用离线 RL 预训练通用 VLA（**π*₀.₆**），然后通过真机数据采集专门化到下游任务。

π*₀.₆ 能叠衣服、组装纸箱、用专业意式咖啡机做咖啡。在最难的任务上，RECAP **吞吐量翻倍**，失败率**减半**。

> **【小白解释】优势条件化 (Advantage Conditioning)**：在模型输入中加一个信号告诉它"当前状态好不好"。训练时：好的动作标 positive，差的标 negative。推理时：让模型专挑 positive 的动作。这和之前 sai_0/π₀ 的纯 BC 完全不同——这是 RL，模型学会了"哪些动作更好"。

## 一、标准 RL 形式化

用标准 RL 设定：智能体策略 `π(a_t|o_t)` 在观测 `o_t` 下选择动作 `a_t`。

轨迹分布：
```
ρ_π(τ) = p(o_0) ∏_{t=0}^{T-1} π(a_t|o_t)·p(o_{t+1}|o_t, a_t)
```

> **【数学推导】ρ_π(τ) 的含义**
> 这是"给定策略 π，产生轨迹 τ 的概率"：
> - 初始观测 p(o₀)
> - 每一步：策略根据观测选动作 π(a_t|o_t)，环境根据动力学转移 p(o_{t+1}|o_t, a_t)
> - 整个序列的概率 = 初始概率 × 所有中间步骤的乘积

**回报**：
```
R(τ) = Σ_{t=0}^{T} r_t
```
(不使用折扣因子，但可轻易添加)

**RL 目标**：
```
J(π) = E_{τ~ρ_π}[R(τ)] = E_{τ~ρ_π}[Σ_{t=0}^{T} r_t]
```

**值函数**：
```
V^π(o_t) = E_{τ_{t+1:T}}[Σ_{t'=t}^{T} r_{t'}]
```

**优势函数**（N 步估计）：
```
A^π(o_t, a_t) = E_{ρ_π}[Σ_{t'=t}^{t+N-1} r_{t'} + V^π(o_{t+N})] - V^π(o_t)
```

> **【小白解释】优势 (Advantage)**：评估"这个动作比平均水平好多少"。
> - 正优势 (>0)：这个动作比预期好 → 增加执行概率
> - 负优势 (<0)：这个动作比预期差 → 减少执行概率
> - 零优势 (=0)：正常 → 保持不变

## 二、正则化 RL 与策略改进定理

正则化 RL 目标（在最大化回报的同时靠近参考策略 π_ref）：

```
J(π, π_ref) = E_{τ~ρ_π}[Σγ^t r_t] - β·E_{o~ρ_π}[D(π(·|o)||π_ref(·|o))]
```

**公式理解**：既要让累积奖励尽量大，又不要让新策略偏离参考策略太远。β 控制偏离的容忍度——β 越小，越激进。

当 D 为 KL 散度时，有解析解：
```
π̂(a|o) ∝ π_ref(a|o) exp(A^{π_ref}(o,a) / β)
```

**优势条件化策略改进定理**（RECAP 基于的核心定理）：

定义改进概率 `p(I|A^{π_ref}(o,a)) = g(A)/∫g(A)da'`，其中 g 是单调增函数，则：
```
π̂(a|o) ∝ π_ref(a|o) · p(I|A^{π_ref}(o,a))^β
```
**保证 J(π̂) ≥ J(π_ref)**（新策略不差于旧策略）。

## 三、分布式值函数训练（核心数学推导）

用分布式值函数 `p_φ(V|o_t, ℓ)` 将观测和语言命令映射到 B=201 个离散桶上的值分布：

```
min_φ E_{τ∈D}[ Σ_{o_t∈τ} H(R^B_t(τ), p_φ(V|o_t, ℓ)) ]
```

**奖励函数定义**（通用稀疏奖励）：
```
r_t = 0,        若 t=T 且成功
      -C_fail,   若 t=T 且失败
      -1,        其他
```

即：值函数学会预测（负的）剩余成功步数。成功 episode 的值 = -(剩余步数)，失败 episode = 大的负数。

> **【数学推导】为什么用这种负步数设计？**
> 越靠近成功，值越大（越接近 0）。同一任务中步骤越少，策略越高效。这让策略不仅追求"成功"，还追求"快"——因为更短的 episode 有更高的总近端奖励。

从分布式值函数中提取连续值和优势：
```
V^{π_ref}(o_t, ℓ) = Σ_{b∈[0,B]} p_φ(V=b|o_t)·v(b)
A^{π_ref}(o_t, a_t, ℓ) = Σ_{t'=t}^{t+N-1} r_{t'} + V^{π_ref}(o_{t+N}) - V^{π_ref}(o_t)
```

## 四、优势条件化策略提取（Bayes 推导）

应用 Bayes 规则重写改进概率：
```
p(I|A^{π_ref}(o,a)) = π_ref(a|I, o) / π_ref(a|o)
```

则改进策略的封闭形式为：
```
π̂(a|o,ℓ) ∝ π_ref(a|o,ℓ) · [π_ref(a|I,o,ℓ) / π_ref(a|o,ℓ)]^β
```

对 β=1 的特例：**π̂(a|o,ℓ) = π_ref(a|I,o,ℓ)**。

改进指示器 I 遵循 delta 分布：
```
p(I|A^{π_ref}(o,a,ℓ)) = δ(A^{π_ref}(o,a,ℓ) > ε_ℓ)
```
其中 ε_ℓ 是任务相关的改进阈值。

> **【小白解释】I 和 ε_ℓ 的作用**
> I 把连续的优势值变成二元的信号：advantage > ε_ℓ → I=True ("这个动作好")。ε_ℓ 控制多"严格"——越高越苛刻。训练时 I=True 表示"这个动作值得模仿"，I=False 表示"别学这个"。

**最终训练目标**（负对数似然最小化）：
```
min_θ E_{D}[ -log π_θ(a_t|o_t,ℓ) - α·log π_θ(a_t|I_t, o_t, ℓ) ]
其中 I_t = 𝟙[A^{π_ref}(o_t, a_t, ℓ) > ε_ℓ]
```

> **【数学推导】两个 log-likelihood 项的含义**
> - 第一项：标准模仿学习——无条件地匹配所有数据
> - 第二项：仅在 I_t=True（好动作）时才最大化——鼓励策略重点模仿"好"动作
> - α 控制第二项（精细化）的相对权重
> - 通过同时训练两项（类似 classifier-free guidance 中的随机丢弃），模型既学会 π_ref(a|o,ℓ) 也学会 π_ref(a|I, o, ℓ)，推理时可以用 CFG 加权组合

对人为纠正：强制 I_t=True（假设专家总是对的）。

## 五、π₀.₆ 架构（从 π₀.₅ 到 π₀.₆ 到 π*₀.₆）

**π₀.₆ 相对于 π₀.₅ 的改进**：
1. 预训练数据加入更多机器人平台数据
2. 基础 VLM 从 PaliGemma 升级到 **Gemma 3 4B**
3. Action Expert 从 300M 升级到 **860M**

**π*₀.₆ = π₀.₆ + 优势条件化**：

在模型输入中加入额外的文本 token：
- `I_t=True` → 输入"Advantage: positive"
- `I_t=False` → 输入"Advantage: negative"

**动作的连续部分的对数似然下界**（Flow Matching + 优势条件化）：

```
log π_θ(a_{t:t+H}, a^ℓ_{t:t+H}|I_t, o_t, ℓ, ℓ̂) ≥
E_{η,ω}[ log p_θ(a^ℓ|I_t, o_t, ℓ, ℓ̂)
         - α_η·||ω - a - f_θ(a^{η,ω}, I_t, o_t, ℓ, ℓ̂)||² ]
```

> **【小白解释】α_η 噪声相关的权重**
> 这个权重项来自推导速度场到对数似然的关系。η 越小（噪声越大）权重越小——模型在"关键去噪步骤"(中等 η)上获得更多训练信号。

训练时随机丢弃 I_t（30%），配合 CFG 实现推理时的 β>1 策略锐化。

## 六、值函数训练效率

值函数使用与 VLA 相同架构但更小的 VLM backbone (**670M Gemma 3**)。在 VLA 训练期间动态计算 advantage，无需额外上训练开销。

## 七、RECAP 算法总结

```
Algorithm: RECAP
1. 在 D_demo 上训练值函数 V_pre (Eq. 1)
2. 在 D_demo 上训练策略 π_pre (Eq. 3, 用 V_pre)
3. 初始化任务数据 D_ℓ (演示)
4. 从 V_pre 微调值函数 V⁰_ℓ (Eq. 1)
5. 从 π_pre 微调策略 π⁰_ℓ (Eq. 3, 用 V⁰_ℓ)
6. for k = 1 to K:
7.   用 π^{k-1}_ℓ 采集数据 → 加入 D_ℓ
8.   从 V_pre 微调值函数 V^k_ℓ (Eq. 1)
9.   从 π_pre 微调策略 π^k_ℓ (Eq. 3, 用 V^k_ℓ)
```

## 八、π₀ vs π₀.₅ vs π₀.₆ 核心区别

| | π₀ | π₀.₅ | π₀.₆ |
|---|---|---|---|
| **方法** | 纯 BC | Co-training | **BC + RL** |
| **目标** | 见过环境高表现 | **全新家庭泛化** | **通过自经验改进** |
| **基础 VLM** | PaliGemma 2B | PaliGemma 2B | **Gemma 3 4B** |
| **Action Expert** | 300M | 300M | **860M** |
| **动作表示** | Flow Matching | FAST 离散 + Flow Matching | FAST + Flow Matching + **优势条件化** |
| **训练方式** | 预训练+后训练 | 预训练+后训练 | **预训练+后训练+在线迭代** |
| **值函数** | ❌ | ❌ | **✅ 670M 分布式值函数** |
| **子任务推理** | 外部 VLM | 模型内部 | ✅ 模型内部 |
| **数据来源** | 10k+ 小时机器人 | +网络、口头指令等（97.6% 非目标机器人） | +自主采集+专家干预 |
| **典型性能提升** | 基线 | 新家庭泛化 ~80%+ | **吞吐量 2×，失败率减半** |

**递进逻辑**：π₀ 证明了 VLM+Flow Matching 能跑复杂任务 → π₀.₅ 证明联合训练让模型能泛化到新环境 → π₀.₆ 证明 RL 能让模型从自己犯的错中学习，在已经能"做任务"的基础上"做得更稳更快"。
