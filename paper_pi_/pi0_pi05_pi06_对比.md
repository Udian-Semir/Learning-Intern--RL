# π₀ vs π₀.₅ vs π₀.₆ 完整对比

## 一、递进关系

```
π₀ (2024)
  → 证明: VLM + Flow Matching 能做到复杂灵巧操作

π₀.₅ (2025)
  → 证明: 联合训练 (co-training) 让模型泛化到全新环境

π₀.₆ (2025)
  → 证明: 强化学习 (RL) 让模型从自己犯的错中学习，做得更稳更快
```

**核心叙事**：π₀ 证明了"能跑"，π₀.₅ 证明了"能在没去过的地方跑"，π₀.₆ 证明了"能越跑越好"。

## 二、架构演变

| | π₀ | π₀.₅ | π₀.₆ |
|---|---|---|---|
| **基础 VLM** | PaliGemma 2B (Gemma 2B) | PaliGemma 2B (Gemma 2B) | **Gemma 3 4B** |
| **Action Expert** | 300M | 300M | **860M** |
| **总参数** | ~3.3B | ~3.3B | **~4.9B** |
| **值函数** | ❌ | ❌ | **✅ 670M Gemma 3 (独立)** |
| **动作生成** | Flow Matching (10 步 Euler) | FAST 离散(预训练) + Flow Matching(后训练) | FAST + Flow Matching + **优势条件化** |
| **子任务推理** | 外部 VLM 策略 | 模型内部自回归 | 模型内部自回归 |

> **【小白解释】VLM 从 2B 升级到 4B 意味着什么？**
> Gemma 3 4B 相比 PaliGemma 2B 参数翻倍，视觉语言理解能力和泛化能力更强。但模型总体参数只从 3.3B→4.9B（增加 50%），因为 Action Expert 的增量（860M-300M=560M）和 VLM 增量（4B-2B=2B）之和有 2.56B。实际只增加 1.6B 的原因是模型复用了更多权重（Gemma 3 本身包含 PaliGemma 的视觉能力，不再需要单独的 SigLIP 视觉编码器）。

## 三、训练流程演变

```
π₀:
  预训练 (10k+ 小时机器人数据)
    → 后训练 (高质量任务数据)
    → 推理

π₀.₅:
  Co-training 预训练 (280k 步, α=0, 离散 token)
    → 后训练 (80k 步, α=10.0, 加 Flow Matching)
    → 推理 (先子任务 ℓ̂ 再动作 a_{t:t+H})

π₀.₆:
  Co-training 预训练 (含价值预训练 V_pre)
    → 后训练 SFT (I=True 固定)
    → 迭代 1: 自采集+干预 → 价值微调 → 策略微调
    → 迭代 K: 同上 → 最终模型
```

## 四、数学公式演变

### π₀：纯 Flow Matching Loss

```
L(θ) = E[||v_θ(A^τ, o_t) - u(A^τ|A_t)||²]
A^τ = τ·A + (1-τ)·ε,  u = A - ε
```

### π₀.₅：联合离散+连续 Loss

```
L(θ) = H(x, f^ℓ_θ) + α·||ω - a - f^a_θ(a^{τ,ω}, o)||²
α=0 预训练, α=10.0 后训练
```

### π₀.₆：优势条件化联合 Loss

```
L(θ) = -log π_θ(a|o) - α·log π_θ(a|I, o)
I = 𝟙[A^{π_ref}(o,a) > ε]
```

**每一步的新增**：
- π₀：建立了 Flow Matching 训练框架
- π₀.₅：在 Flow Matching 基础上，增加离散 token 联合训练（加快预训练）
- π₀.₆：在联合训练基础上，增加优势条件化（让 RL 信号指导策略改进）

## 七、核心创新总结

| 版本 | 核心创新 | 解决了什么问题 |
|------|---------|-------------|
