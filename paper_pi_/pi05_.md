## 摘要

为了机器人真正有用，它们必须在真实世界中执行实际相关的任务。视觉-语言-动作模型 (VLA) 展示了令人印象深刻的结果，但它们能泛化到什么程度仍是开放问题。

我们描述了 **π₀.₅**，一个基于 π₀ 的新模型，通过在异构任务上联合训练 (co-training) 实现广泛泛化。π₀.₅ 使用多种机器人数据、高层语义预测、网络数据等，实现可泛化的真实世界操作。

**核心贡献**：首次证明端到端学习式机器人系统可在**全新家庭**中执行长时间灵巧操作（清洁厨房或卧室）。

> **【小白解释】联合训练 (Co-training)**：同时在多种不同类型的任务上训练同一个模型。例如一边学"抓苹果"，一边学"给图片写描述"。这让模型既学会物理操作又保留语言理解力。

## 一、分层架构设计

π₀.₅ 分布分解：

```
π_θ(a_{t:t+H}, ℓ̂|o_t, ℓ) = π_θ(a_{t:t+H}|o_t, ℓ̂) · π_θ(ℓ̂|o_t, ℓ)
```

- `a_{t:t+H}`: H 步动作块
- `ℓ̂`: 模型预测的下一步子任务（如"拿起盘子"）
- `o_t`: 观测（图像+关节状态）
- `ℓ`: 高层任务指令（如"清理厨房"）

**推理流程**：每步先预测子任务 ℓ̂（高层推理），再基于 ℓ̂ 预测动作（低层执行）。同一模型完成两个层次。

## 二、联合损失函数（核心数学推导）

Flow Matching 噪声动作：

```
a^{τ,ω}_{t:t+H} = τ·a_{t:t+H} + (1-τ)·ω,   ω ~ N(0,I)
```

**联合 Loss**：

```
min_θ E[ H(x_{1:M}, f^ℓ_θ(o_t, ℓ)) + α·||ω - a_{t:t+H} - f^a_θ(a^{τ,ω}, o_t, ℓ)||² ]
```

- 第一项: 离散文本 token 的交叉熵（含 FAST 动作 token）
- 第二项: Flow Matching 的 MSE 损失
- α 权衡参数（预训练 0，后训练 10.0）

> **【数学推导】为什么 α=0 预训练更快？**
> 离散 token 训练（B=32 批量、280k 步）比 Flow Matching 训练快约 3-5×，因为不需要 10 步前向传过去噪。280k 步 × 快 3× ≈ 相当于 840k 步的连续训练等效量。这就是"先用离散 token 预训练、再加 Flow Matching 后训练"策略的核心动机。

**预训练**：纯自回归 Transformer，α=0，280k 步
**后训练**：加 Action Expert（随机初始化），α=10.0，80k 步
**推理**：自回归解码 ℓ̂ → 10 步 Flow Matching 去噪生成动作

## 三、五大训练数据源

| 数据 | 简称 | 内容 |
|------|------|------|
| 移动操作机器人 | MM | ~400 小时，~100 家庭 |
| 多环境非移动机器人 | ME | 单/双臂，各种家庭 |
| 跨本体实验室 | CE | 实验室条件下多种任务 |
| 高层子任务预测 | HL | 人工标注语义子任务 |
| 网络数据 | WD | 图像描述、QA、物体定位 |
| 口头指令 | VI | 人用语言"遥控"机器人 |

**关键数据**：**97.6%** 的训练数据不来自目标移动操作机器人！这证明知识迁移的力量。

---

## 四、消融实验

| 移除项 | 影响 |
|--------|------|
| no ME | 跨环境迁移被削弱 → 性能显著下降 |
| no CE | 跨任务迁移被削弱 → 性能显著下降 |
| no WD | 语言跟随下降（尤其未见过的物体类别）|
| no VI | 口头指令数据仅占 11%，但移除后性能显著下降 |

## 五、与 π₀ 核心区别

| | π₀ | π₀.₅ |
|---|-----|-------|
| 泛化目标 | 见过环境 | **全新家庭** |
| 训练数据 | 10k+ 小时机器人 | 400h MM + 97.6% 非机器人 |
| 高层推理 | 外部 VLM 策略 | **模型内部分层推理** |
| 动作表示 | 纯 Flow Matching | **FAST 离散 + Flow Matching** |
| 联合训练 | 仅机器人动作 | +子任务 +网络 +口头指令 |
